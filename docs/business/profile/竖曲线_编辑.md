# 竖曲线编辑

> 文档粒度要求：本文只描述竖曲线 WPF 参数编辑和回写桥接，不描述创建命令或夹点/PVI 命令细节。

## 基本信息

- 功能名称：竖曲线编辑
- 所属模块：纵断面设计
- 命令名称：`RD_PROFILE_VERTICAL_CURVE_EDIT_HANDLE`、`RD_PROFILE_VERTICAL_CURVE_APPLY_DIALOG_FILE`
- 托管弹窗命令：`RD_PROFILE_VERTICAL_CURVE_SHOW_WPF_DIALOG`
- 对应代码入口：`ObjectArxProfileVerticalCurveCommand.*`、`ProfileVerticalCurveDialogBridge.*`、`ProfileVerticalCurveWindow.xaml`
- 业务文档维护人：RoadProto
- 原型版本：V0.1.9 follow-up
- 是否可复用：是

## 功能背景

竖曲线需要支持双击后弹出参数窗口，修改起终点、PVI 和半径。RoadProto 当前采用文件 Bridge：C++ 写请求文件，AutoCAD .NET 托管命令打开 WPF，WPF 写响应文件，再由 C++ 读取并调用 application/domain 层完成校验和回写。

## 业务目标

提供不依赖具体 UI 技术栈的竖曲线参数编辑流程。当前 WPF 窗口支持编辑名称、起终点桩号/高程、通过左右按钮切换 PVI，并编辑当前 PVI 的桩号、高程和半径。

## 输入条件

- CAD 对象：`DnProfileVerticalCurveEntity` 的 handle。
- 用户输入参数：WPF 窗口中的名称、起终点、PVI 桩号/高程、半径。
- 外部数据：临时请求/响应文件。

## 输出结果

- 更新 `ProfileVerticalCurveData`。
- 重新计算控制点、BVC/EVC、曲线采样和高低点信息。
- 刷新 `DnProfileVerticalCurveEntity` 图形。

## 操作流程

1. 用户双击竖曲线实体，托管 Ribbon 插件识别 `DNPROFILEVERTICALCURVEENTITY`，发送 `RD_PROFILE_VERTICAL_CURVE_EDIT_HANDLE <handle>`。
2. C++ 命令打开实体并生成 `ProfileVerticalCurveDialogRequest`。
3. `ProfileVerticalCurveDialogBridge` 写请求文件和 pending 文件，发送 `RD_PROFILE_VERTICAL_CURVE_SHOW_WPF_DIALOG`。
4. WPF 读取请求文件并打开 `ProfileVerticalCurveWindow`。
5. 用户确定后 WPF 写响应文件，并发送 `RD_PROFILE_VERTICAL_CURVE_APPLY_DIALOG_FILE "<responsePath>"`。
6. C++ 严格解析响应文件，调用 `ProfileVerticalCurveEditService::applyDialogEdit()` 校验并写回实体。

## 关键业务规则

- 终点桩号必须大于起点桩号。
- PVI 半径必须大于等于 `1`。
- PVI 桩号必须位于起终点之间，且 PVI 桩号不能重复。
- 响应文件中的数值字段必须完整且可解析为有限数；解析失败不回写实体。
- WPF 只负责 DTO 展示和基础输入校验，不承载竖曲线核心计算。
- C++ application/domain 层负责最终校验、重建和错误恢复。

## 可复用性说明

- `ProfileVerticalCurveDialogBridge` 可复用为后续竖曲线参数窗口、要素表窗口和批量编辑窗口的文件桥接模式。
- `ProfileVerticalCurveEditService` 可复用于 WPF、命令行、夹点和未来 Palette/MFC/Qt UI。
- 当前 Bridge 为原型文件协议，后续可替换为更正式的进程内 Bridge，但 DTO 字段应保持 UI 无关。

## 后续扩展

- 在窗口中显示 BVC/EVC、高点/低点、坡度差、长度和 K 值。
- 支持新增/删除 PVI 的窗口按钮。
- 支持非对称竖曲线参数。

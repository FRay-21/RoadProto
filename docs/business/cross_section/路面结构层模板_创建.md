# 路面结构层模板_创建

## 基本信息

- 功能名称：路面结构层模板创建
- 所属模块：横断面设计
- 命令名称：`RD_SECTION_PAVEMENT_LAYER_TEMPLATE_CREATE`
- 对应代码入口：`src/cad_adapter/objectarx/cross_section/ObjectArxPavementLayerTemplateCommand.cpp`
- 业务文档维护人：RoadProto 项目
- 原型版本：`v0.1.19`
- 是否可复用：部分可复用

## 功能背景

横断面道路模型需要把路基模板部件与可复用的路面结构层参数分离。路面结构层模板作为独立 DWG 实体保存层类型、厚度、内外侧加宽和横坡信息，供多个路基模板部件引用，并可通过 `.rpavement.xml` 在图纸或项目间流转。

## 业务目标

- 在 CAD 图中创建独立 `DnPavementLayerTemplateEntity`。
- 通过 WPF 参数窗口维护模板名称、显示比例、预览宽度和结构层列表。
- 支持结构层类型下拉值：上面层、中面层、下面层、基层、底基层、垫层。
- 支持“内外厚度是否一致”复选框；勾选时使用单一厚度，未勾选时分别填写内侧厚度和外侧厚度。
- 明确内外侧语义：内侧是相对路基部件更靠近道路中线的一侧，外侧是更远离道路中线的一侧；内侧加宽表示向道路中线方向扩宽。
- 支持 WPF 窗口导入/导出 `.rpavement.xml`。

## 适用场景

适用于需要在横断面路基模板中复用同一套路面结构层的原型图纸。模板实体本身不绑定道路中线，可被任意路基模板部件引用。

## 输入条件

- CAD 选择对象：无，用户只点取模板插入点。
- 用户输入参数：模板名称、显示比例、预览宽度、结构层类型、层名、厚度、内外侧加宽和内外侧坡度。
- 已有设计实体：无。
- 外部数据：可选 `.rpavement.xml`。

## 输出结果

- CAD 图形实体：`DnPavementLayerTemplateEntity`
- 领域实体：`PavementLayerTemplateData`
- 表格或报告：无
- 更新通知或重建请求：当前版本不自动触发引用方重建。

## 操作流程

1. 运行 `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_CREATE` 或点击 Ribbon `横断面设计 / 创建路面结构层模板`。
2. 用户在 CAD 图中点取模板插入位置。
3. C++ 应用服务生成默认 `PavementLayerTemplateData`。
4. ObjectARX 命令创建 `DnPavementLayerTemplateEntity`，写入插入点和默认数据，并追加到模型空间。
5. 命令读取新实体 handle，将 handle、插入点和模板数据写入 WPF 请求文件。
6. WPF 打开路面结构层模板窗口；用户可编辑层列表，也可导入 `.rpavement.xml` 覆盖当前窗口数据。
7. 用户确认后，WPF 写出响应文件并发送 `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_APPLY_DIALOG_FILE`。
8. C++ 回写命令按 handle 更新刚创建的模板实体。

## 关键业务规则

- 路面结构层类型只能使用：上面层、中面层、下面层、基层、底基层、垫层；桥接和 XML 中使用稳定枚举编码保存。
- 每一层必须有正厚度；等厚模式保存 `thickness`，非等厚模式保存 `innerThickness` 和 `outerThickness`。
- 内侧和外侧按路基部件解释，不按 CAD 图纸左/右绝对方向解释。
- 内侧加宽扩向道路中线，外侧加宽扩向远离道路中线的一侧。
- WPF 只负责参数编辑、预览和 XML 文件读写；模板数据归一化由 C++ `PavementLayerTemplateRules` 执行。
- `.rpavement.xml` 用于模板参数流转，不代表 DWG 实体持久化格式。

## 可复用性说明

- 可复用内容：`PavementLayerTemplateModel`、`PavementLayerTemplateRules`、`PavementLayerTemplateCreateService`、`DnPavementLayerTemplateEntity`
- 临时原型内容：WPF 请求/响应文件 Bridge、当前命令中的插入点点取和弹窗调起流程
- 正式复用前需要改造的内容：模板库、结构层材料库、统一实体关系和引用方自动重建

## 与其他模块的依赖关系

- 上游模块：无
- 下游模块：横断面路基模板、横断面道路模型、查看横断面
- 实体联动行为：当前版本由路基模板部件保存结构层模板 handle，道路模型生成时读取；模板更新后需要重新生成道路模型。

## 后续对接正式 EICAD 功能的注意事项

- 后续应增加材料、压实度、造价、分层编号等正式结构层属性。
- 后续应接入统一关系管理机制，提示引用该模板的路基模板或道路模型需要重建。

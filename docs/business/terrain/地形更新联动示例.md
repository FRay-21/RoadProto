# 地形更新联动示例

## 基本信息

- 功能名称：地形数模更新后的联动标记示例
- 所属模块：地形数模
- 命令名称：`RD_TERRAIN_MARKDIRTY`
- 对应代码入口：`TerrainUpdateSampleService.*`、`TerrainModule.cpp`
- 业务文档维护人：RoadProto
- 原型版本：`v0.1.6` 文档同步；命令最初引入于 `v0.1.0`
- 是否可复用：部分可复用

## 当前定位

本文件是地形模块的“关系联动框架示例”，不是当前 TIN 地形数模的主业务文档。当前地形构网见 `docs/business/terrain/地形构网_TIN地形数模.md`，地形数模编辑见 `docs/business/terrain/地形数模_编辑.md`，`.rmesh` 流转见 `docs/business/terrain/地形数模_RMesh导入导出.md`。

V0.1.6 中，`RD_TERRAIN_MARKDIRTY` 仍保留为最小示例，用来说明 RoadProto 后续如何通过统一关系管理机制表达“地形更新后，下游纵断面、横断面、土方等对象需要重建”。它不代表 `DnTerrainTinEntity` 已完成自动监听源对象修改，也不代表真实道路设计对象已经完成地形联动计算。

## 功能背景

道路设计中，纵断面地面线、横断面地面线、土方计算等内容通常依赖地形数模。当地形数据变化时，下游对象需要被标记为待更新。V0.1 不计算真实地面线，只验证统一关系管理和脏标记传播是否可用。

## 业务目标

通过一个最小示例证明：地形数模更新后，可以由 `EntityRelationManager` 统一生成下游实体的重建请求，而不是由地形模块直接调用纵断面或横断面模块。

## 适用场景

- 框架加载和命令注册验证。
- 后续 `DnTerrainTinEntity::RebuildFromSourceObjects`、源对象 reactor 和道路对象重建队列的设计参考。
- 后续纵断面、横断面地面线刷新流程的前置机制验证。

## 输入条件

- CAD 选择对象：V0.1 暂无。
- 用户输入参数：V0.1 暂无。
- 已有设计实体：内存中的示例实体。
- 外部数据：暂无。

## 输出结果

- 领域实体：
  - `terrain.sample.tin.001`
  - `profile.sample.groundline.001`
  - `cross_section.sample.groundline.001`
- 更新结果：
  - 纵断面地面线标记为 `NeedsRebuild`
  - 横断面地面线标记为 `NeedsRebuild`
- CAD 图形实体：V0.1 暂无。

## 操作流程

1. 命令回调触发 `TerrainUpdateSampleService`。
2. 服务确保示例地形、纵断面、横断面实体存在。
3. 服务记录纵断面和横断面对地形的依赖关系。
4. 服务将地形实体标记为已更新。
5. `EntityRelationManager` 将下游实体标记为需要重算和刷新图形。
6. 命令通过 editor adapter 输出执行摘要。

## 关键业务规则

- 依赖关系必须通过 `EntityRelationManager` 表达。
- 地形源实体更新后保持 `Clean`。
- 被影响实体变为 `NeedsRebuild`。
- V0.1.6 只保留框架示例和重建请求，不执行真实道路计算，也不自动重建当前 `DnTerrainTinEntity`。

## 可复用性说明

- 可复用内容：
  - `EntityRelationManager`
  - `DesignEntityDescriptor`
  - `UpdateEvent`
  - `RebuildRequest`
  - application service 组织方式
- 临时原型内容：
  - 固定示例 ID
  - 只在内存中保存关系
  - 未做 DWG 持久化

## 与其他模块的依赖关系

- 上游模块：地形数模。
- 下游模块：纵断面、横断面。
- 实体联动行为：地形更新后，纵断面和横断面示例实体需要重算并刷新图形。
- 当前 TIN 实体联动状态：`DnTerrainTinEntity` 已保存来源 handle、文字 handle、块名、属性 Tag，并预留 `m_isDirty` 和 `RebuildFromSourceObjects`；V0.1.6 尚未实现源对象修改 reactor，也不会在源图元变化后自动更新数模。

## 后续对接正式 EICAD 功能的注意事项

- 用持久化实体 ID 和 AutoCAD handle 替换固定示例 ID。
- 增加地形对象选择和地形元数据保存。
- 在增加真实地面线计算前，先完成关系持久化和重建队列。

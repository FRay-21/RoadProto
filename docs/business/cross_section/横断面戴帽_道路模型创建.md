# 横断面戴帽_道路模型创建

## 基本信息

- 功能名称：横断面戴帽道路模型创建
- 所属模块：横断面设计
- 命令名称：`RD_SECTION_ROAD_MODEL_CREATE`
- 对应代码入口：`src/cad_adapter/objectarx/cross_section/ObjectArxRoadModelCommand.cpp`
- 业务文档维护人：RoadProto 项目
- 原型版本：`v0.1.19`
- 是否可复用：部分可复用

## 功能背景

横断面戴帽用于把道路中线、纵断面竖曲线设计高程、已有路基模板和边坡模板按桩号范围组合，生成三维道路模型。该功能验证横断面模板优先级、桩号范围裁剪、边坡交地搜索、采样断面节点链、道路三维网格线框和结构层弱化填充面显示流程。

## 业务目标

- 选择一条道路中线作为平面路径。
- 自动匹配同一中线关联的竖曲线；无法唯一匹配时由用户选择竖曲线。
- 通过 WPF 表格配置路基模板生效范围，并可在每一行从图中点选路基模板实体。
- 通过 WPF `边坡模板` tab 分别配置左侧、右侧模板组、模板优先级和搜索宽度。
- 在边坡模板组行内提供 `管理模板组` 入口，便于维护组内多模板的搜索顺序。
- 读取路基模板部件绑定的路面结构层模板，并在道路模型中生成结构层三维边界线和弱化填充面。
- 生成道路模型时在 AutoCAD 状态栏显示进度，避免大图纸生成过程无反馈。
- 按模板优先级和采样规则创建 `DnRoadModelEntity` 三维道路模型实体。
- 保存生成时使用的采样桩号、每个采样断面的结构层节点和地面剖面快照，供后续 `查看横断面` 功能按生成时状态快速切换断面。

## 适用场景

适用于已有道路中线、纵断面竖曲线和路基模板实体的原型图纸。当前版本的路基和边坡仍以三维线框表达；路面结构层在 `DnRoadModelEntity` 中按模板轮廓显示弱化填充面和边线，但不生成可算量的实体体积或材质对象。

## 输入条件

- CAD 选择对象：`DnRoadCenterlineEntity`
- 用户输入参数：采样间距、起终点桩号、路基模板点选结果、边坡模板点选结果、模板 handle、模板优先级行顺序、左右搜索宽度
- 已有设计实体：`DnProfileVerticalCurveEntity`、`DnSubgradeTemplateEntity`、`DnSlopeTemplateEntity`、部件绑定的 `DnPavementLayerTemplateEntity`、可选 `DnTerrainTinEntity`
- 外部数据：无

## 输出结果

- CAD 图形实体：`DnRoadModelEntity`
- 领域实体：`RoadModelData`，包含三维部件线、边坡线、路面结构层边界线、采样断面节点链、采样断面结构层节点、采样断面地面剖面快照、网格线框和采样桩号
- 表格或报告：无
- 更新通知或重建请求：当前版本不自动接入统一关系重建

## 操作流程

1. 运行 `RD_SECTION_ROAD_MODEL_CREATE` 或点击 Ribbon `横断面设计 / 横断面戴帽`。
2. 选择道路中线实体。
3. 系统查找与该中线关联的唯一竖曲线；不唯一时提示用户选择竖曲线。
4. 系统校验竖曲线所属拉坡图确实关联当前道路中线。
5. 打开 WPF `横断面戴帽` 窗口，在 `路基模板` tab 中编辑模板范围表。
6. 用户可在某一行点击 `点选`，回到 CAD 图中选择 `DnSubgradeTemplateEntity`，系统把模板 handle 和名称回填到该行后重新打开窗口。
7. 用户可在 `边坡模板` tab 的左侧或右侧页面中新增模板组，并通过 `管理模板组` 选中当前组后点选、删除、上移、下移或清空组内 `DnSlopeTemplateEntity`。
8. 点击 `生成模型`。
9. C++ ObjectARX 读取回写文件，解析路基模板、边坡模板、结构层模板、搜索宽度和模板组配置并调用 `RoadModelBuildService`；构建期间通过状态栏进度条显示校验、采样、生成和整理阶段。
10. 系统创建 `DnRoadModelEntity`，在 CAD 中绘制三维道路模型；结构层先按四边形/梯形轮廓绘制弱化填充面，再叠加层色边线。

## 关键业务规则

- 表格行越靠上优先级越高；重叠桩号范围采用优先级最高的模板。
- 行内点选模板只负责回填当前行的模板 handle 和模板名称，不在 WPF 中直接读取或修改 CAD 实体。
- 模板范围可短于或长于道路中线，生成时只取道路中线、竖曲线和模板范围的有效交集。
- 未命中任何模板的桩号范围不生成道路模型线。
- 采样点包含道路中线端点、模板边界、竖曲线关键点和用户设置的采样间距点。
- 每个采样桩号会生成一组断面节点链，节点覆盖路基模板边界、边坡模板边界和交地点；同一桩号还保存左右侧地面剖面快照，`DnRoadModelEntity` 按三维坐标和剖面快照持久化。
- 路基模板部件如果保存了路面结构层模板 handle，道路模型生成时必须读取对应 `DnPavementLayerTemplateEntity`；所有部件类型均允许绑定结构层模板。
- 结构层内外侧语义沿用模板定义：内侧是当前路基部件更靠近道路中线的一侧，外侧是更远离道路中线的一侧；左侧和右侧道路均按该语义翻译为横断面偏距方向。
- 结构层等厚时按单一厚度生成；非等厚时按内侧厚度和外侧厚度生成。内侧正加宽向道路中线方向扩展，外侧正加宽向远离中线方向扩展；负加宽表示当前层顶边相对上一层底边所在直线缩短。
- 结构层内外侧坡度只作用于当前层自身侧边：除第一层外，当前层顶边以上一层底边所在直线为基准，加宽沿该直线平行/共线延长或收回，底边再按厚度和 `1:n` 坡度计算水平位移。坡度为 0 时顶边到底边的侧边竖直，正坡度向外放，负坡度向内收；坡度输入按路面结构层模板规则支持 `1:n` 或数字 `n`。
- 结构层道路模型必须与路面结构层模板预览采用同一组四边形/梯形轮廓：每层只输出顶边、底边、内侧边和外侧边，不得保留六点台阶表达，避免模型、模板实体和 WPF 预览样式分叉。
- 结构层边界线作为 `RoadModelWireLineKind::PavementLayer` 写入道路模型；`DnRoadModelEntity` 优先从连续 `pavementLayerLines` 组合顶边、底边、内侧边和外侧边，生成每段结构层的顶面、底面、内外侧面和端面弱化填充，再叠加 `wireLines` 线框。没有 `pavementLayerLines` 的旧数据才回退到采样断面节点。
- 道路模型线框由断面横向肋线、相邻断面的纵向连接线、最外侧边界闭合线、起终点端部封闭线和断面节点数量不一致时的过渡线组成。
- 无模板命中的桩号空档不会生成跨空档的纵向连接线；连续模板切换处允许通过过渡线连接。
- 断面节点数量不一致时，路基公共节点按同侧对应位置纵向连接，过渡线只在公共外缘之后的变化尾部生成，避免边坡过渡线进入路基模板内部。
- 路基/道路主体颜色使用现有路基模板部件颜色；边坡线颜色来自边坡模板部件自带颜色，不按填方/挖方强制覆盖；结构层填充面和边线颜色使用路面结构层模板层保存 RGB，不继承路基部件颜色。
- 保留三维部件线和边坡线作为兼容数据；当 `RoadModelData.wireLines` 存在时，`DnRoadModelEntity` 先绘制结构层填充面，再按网格线框绘制可见边。
- `DnRoadModelEntity` 会保存本次生成使用的采样桩号；`查看横断面` 命令优先使用该桩号列表生成断面预览。
- 地面剖面快照随道路模型生成/刷新一起更新，表示生成模型当时的 TIN 地面状态；若后续 TIN 变化，应重新生成道路模型以刷新快照。
- TIN 横断面剖切在领域层先建立临时 flat 网格候选索引，当前断面线只遍历线段穿越格子内的候选三角形，再用原有边-剖面线相交逻辑精确求交；索引只服务本次生成，不写入 `DnRoadModelEntity`，也不作为长期内存缓存。
- 左、右边坡搜索宽度以道路中线为基准，默认 `50`。
- 边坡模板组上方优先，组内模板上方优先；先戴帽成功的模板生效。
- 进度回调只表达生成状态，不改变模板优先级、采样结果或戴帽成功判定。

## 可复用性说明

- 可复用内容：`RoadModelBuilder`、`RoadModelStationSampler`、`RoadModelTemplateResolver`、`RoadModelSlopeTemplateGroupResolver`、`RoadModelBuildService`、路面结构层模板收集与生成规则、`DnRoadModelEntity`
- 临时原型内容：WPF 请求/响应文件桥接、当前命令中的 AutoCAD 选择流程
- 正式复用前需要改造的内容：接入统一实体关系管理、自动重建、模板库选择器和可参与算量/材质的正式三维实体模型

## 与其他模块的依赖关系

- 上游模块：平面设计、纵断面设计、横断面路基模板、路面结构层模板
- 下游模块：三维模型、出图、算量和联动重建
- 实体联动行为：当前版本保存 handle 关联，不自动监听上游实体变更

## 后续对接正式 EICAD 功能的注意事项

- 当前已支持从图中点选模板实体，后续仍需要补充模板库选择和更明确的错误定位提示。
- 需要补充道路面、可编辑可算量的结构层实体体积、边坡面、排水等对象的正式三维表达和算量属性。
- 需要接入统一关系管理机制，使中线、竖曲线或模板变更后可提示或自动重建道路模型。

# 平面布线 / 道路中线创建

## 基本信息

- 功能名称：道路中线创建
- 所属模块：平面设计
- 命令名称：`RD_ALIGN_CENTERLINE_CREATE`
- 对应代码入口：`ObjectArxAlignmentCommand.cpp`、`AlignmentCreateService.*`、`HorizontalAlignmentBuilder.*`
- 业务文档维护人：RoadProto
- 原型版本：v0.1.8
- 是否可复用：是

## 功能背景

道路中线是纵断面、横断面、交叉口、地形取高和出图算量的基础对象。本功能验证从 CAD 图面点取控制点生成道路中线实体的最小闭环。

## 业务目标

- 用户点取起点、交点和第三点后，生成直线、缓和曲线、圆曲线、缓和曲线、直线五单元道路中线。
- 用户继续点取后续控制点时，按控制点链生成多组连续五单元平曲线。
- 创建过程中图面显示道路中线动态预览，确认后写入 `DnRoadCenterlineEntity`。

## 适用场景

- 道路平面线形原型验证。
- 后续纵断面、横断面和交叉口设计前的路线基础对象创建。
- CAD 图纸中暂无正式路线对象，需要从人工点取控制点开始建立路线。

## 输入条件

- CAD 选择对象：无。用户按命令提示点取控制点。
- 用户输入参数：起点、交点、第三点和后续控制点；默认半径、缓和曲线长度来自当前原型默认值或后续参数编辑。
- 已有设计实体：无。
- 外部数据：无。

## 输出结果

- CAD 图形实体：`DnRoadCenterlineEntity`。
- 领域实体：`HorizontalAlignment`、道路中线控制点、平曲线要素、特征点和桩号。
- 表格或报告：无。
- 更新通知或重建请求：创建后暂不触发其他模块重建。

## 操作流程

1. 用户运行 `RD_ALIGN_CENTERLINE_CREATE` 或点击 Ribbon `平面布线`。
2. 用户依次点取起点、交点和第三点。
3. 第三点确认后，系统生成道路中线预览实体并插入 DWG。
4. 用户可继续点取后续控制点；每新增一个控制点，系统按控制点链重建当前道路中线。
5. 用户按 `Enter`、鼠标右键或 `ESC` 结束点取。
6. 系统打开道路中线属性窗口，用户确认道路名称、道路等级、数模关联和桩号标注间距。

## 关键业务规则

- 缓和曲线采用回旋线，满足 `rho * l = A^2` 与 `A^2 = R * Ls`。
- 交点按前后切线延长线交点计算，作为五单元主点计算的 PI。
- 道路中线分元素显示：直线段青色，第一缓和曲线红色，圆曲线黄色，第二缓和曲线洋红色。
- 起点桩号默认为 `K0+000`。
- 桩号标注间距默认 `100m`，图中按引线标注形式绘制。
- 道路名称默认按 `K1`、`K2`、`K3` 递增。
- 无效半径、负缓和曲线长或切线空间不足时不得写入实体。
- 后续点取阶段不显示 AutoCAD 自带橡皮筋直线，也不显示上一轮实体末端残留短直线。

## 可复用性说明

- 可复用内容：回旋线计算、五单元平曲线构建、连续控制点链构建、桩号格式化、道路中线实体创建流程。
- 临时原型内容：创建结束后立即打开 WPF 属性窗口的交互编排。
- 正式复用前需要改造的内容：规范参数推荐、控制点编辑模式、设计速度和道路等级驱动的默认参数。

## 与其他模块的依赖关系

- 上游模块：无。
- 下游模块：道路中线属性编辑、平曲线参数编辑、夹点动态编辑、纵断面、横断面、交叉口。
- 实体联动行为：第一版仅创建道路中线实体，后续再接统一关系管理。

## 后续对接正式 EICAD 功能的注意事项

- 创建算法应继续沉淀在 C++ 领域层，不进入 WPF。
- CAD 点取、Jig 预览和实体插入应继续放在 ObjectARX 适配层。
- 后续可增加路线命名规则、起点桩号输入、断链输入和规范参数推荐。

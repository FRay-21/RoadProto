# 边坡模板_编辑

## 基本信息

- 功能名称：边坡模板编辑
- 所属模块：横断面设计
- 命令名称：`RD_SECTION_SLOPE_TEMPLATE_EDIT_HANDLE`
- 对应代码入口：`src/cad_adapter/objectarx/cross_section/ObjectArxSlopeTemplateCommand.cpp`
- 业务文档维护人：RoadProto 项目
- 原型版本：`v0.1.14`
- 是否可复用：部分可复用

## 功能背景

边坡模板创建后，需要允许用户重新打开参数窗口，调整部件顺序、坡率、坡高、宽度、搜索增值和控制条件，并刷新同一个 CAD 自定义实体。

## 业务目标

- 支持通过双击或 handle 命令编辑已有 `DnSlopeTemplateEntity`。
- 保留实体中保存的名称、显示比例、模板类型、部件列表和控制条件。
- 编辑确认后刷新原实体，不创建新实体。

## 操作流程

1. 双击边坡模板实体，托管 Ribbon 事件转发 `RD_SECTION_SLOPE_TEMPLATE_EDIT_HANDLE <handle>`。
2. C++ 通过 handle 打开 `DnSlopeTemplateEntity` 并读取保存数据。
3. 系统打开 WPF `边坡模板` 参数窗口。
4. 用户修改模板参数、部件参数或控制条件。
5. 用户确认后，WPF 写回响应文件。
6. C++ 校验并写回原实体，图形按新参数重绘。

## 关键业务规则

- 编辑命令只能更新选中的边坡模板实体。
- 如果 handle 无效或对象类型不是 `DnSlopeTemplateEntity`，命令应拒绝继续。
- 部件顺序按列表顺序保存，横断面戴帽使用同一顺序执行放坡。
- 无效几何参数不应写入实体，避免后续道路模型构建得到不可解释模板。

## 当前限制

- 自动化测试不覆盖真实鼠标双击手感，仍需要在 AutoCAD 图形界面人工点验。
- 当前不支持把边坡模板绑定到具体道路中线或桩号范围；范围配置在横断面戴帽中完成。

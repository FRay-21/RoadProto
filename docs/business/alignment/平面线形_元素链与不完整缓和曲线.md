# 平面线形 / 元素链与不完整缓和曲线

## 基本信息

- 功能名称：元素链与不完整缓和曲线
- 所属模块：平面设计
- 命令名称：无。当前为领域层内部能力。
- 对应代码入口：`AlignmentElementChainBuilder.*`、`AlignmentGeometry.h`
- 业务文档维护人：RoadProto
- 原型版本：v0.1.8
- 是否可复用：是

## 功能背景

五单元平曲线不能覆盖所有道路平面线形表达。ICD `5/6`、卵形曲线、S 型曲线和后续 C 型曲线都需要用更通用的线形元素链表达。

## 业务目标

- 支持不依赖控制点五单元模型的内部线形元素链。
- 第一版支持圆曲线、有限半径到有限半径的不完整缓和曲线、圆曲线连续组合。
- 支持计算、采样、显示、DWG 保存和作为 ICD 导入导出的内部表达。

## 适用场景

- ICD `5/6` 不完整缓和曲线导入。
- 两个同向圆曲线之间用缓和曲线相接的卵形曲线。
- 正负曲率之间过渡的 S 型曲线。
- 后续扩展 C 型曲线和更多积木法线形组合。

## 输入条件

- CAD 选择对象：无。
- 用户输入参数：当前无直接交互入口。
- 已有设计实体：可由 ICD 导入或后续功能构造。
- 外部数据：可来自 ICD 文件。

## 输出结果

- CAD 图形实体：可保存到 `DnRoadCenterlineEntity` 的元素链道路中线。
- 领域实体：`HorizontalAlignment`、`HorizontalAlignmentElement`、采样点、特征点和桩号。
- 表格或报告：无。
- 更新通知或重建请求：无。

## 操作流程

1. 上游能力构造 `AlignmentElementChainInput`。
2. 输入起点、起始方位角、起点桩号和线形元素集合。
3. 元素链构建器按元素顺序计算采样点、起终点、起终方位角和起终曲率。
4. 构建结果写入 `HorizontalAlignment`。
5. `DnRoadCenterlineEntity` 负责显示、桩号和 DWG 持久化。

## 关键业务规则

- 不完整缓和曲线采用曲率线性变化思想。
- 有限半径到有限半径过渡时，起终曲率不得相同。
- 同向有限曲率过渡可表达卵形曲线。
- 正负曲率过渡可表达 S 型曲线。
- 当前 `CCurve` 作为组合类型预留，后续可扩展为圆曲线、完整缓和曲线、完整缓和曲线、圆曲线组合。
- 元素链实体第一版不提供夹点编辑、WPF 参数编辑或 Ribbon 独立入口。

## 可复用性说明

- 可复用内容：元素链输入模型、圆曲线采样、不完整缓和曲线采样、组合类型标识、DWG V2 持久化数据。
- 临时原型内容：当前仅由 ICD 导入等内部流程使用。
- 正式复用前需要改造的内容：补充交互编辑入口、参数校核、规范推荐和更多线形组合类型。

## 与其他模块的依赖关系

- 上游模块：ICD 导入、后续复杂路线编辑。
- 下游模块：道路中线实体显示、ICD 导出、纵断面和横断面。
- 实体联动行为：第一版无外部联动。

## 后续对接正式 EICAD 功能的注意事项

- 元素链应作为后续路线通用表达之一，不应绑定到某个 UI。
- 后续新增 C 型、卵形和 S 型交互编辑时，应继续复用该领域模型。

# 道路中线 / 属性编辑

## 基本信息

- 功能名称：道路中线属性编辑
- 所属模块：平面设计
- 命令名称：`RD_ALIGN_CENTERLINE_EDIT_HANDLE`
- 对应代码入口：`AlignmentDialogBridge.*`、`AlignmentCenterlineWindow.xaml`、`AlignmentDialogCommands.cs`
- 业务文档维护人：RoadProto
- 原型版本：v0.1.8
- 是否可复用：部分可复用

## 功能背景

道路中线创建后需要维护道路名称、道路等级、数模关联和桩号标注间距。该功能提供双击实体或创建后自动打开的属性编辑入口。

## 业务目标

- 用户可查看和修改道路中线基础属性。
- 双击 `DnRoadCenterlineEntity` 时按实体 handle 打开同一个属性窗口。
- 只修改非几何属性时不触发道路中线几何重建。

## 适用场景

- 创建道路中线后确认基础属性。
- 后续需要修改道路名称、道路等级、数模关联或桩号标注间距。
- 从 DWG 重开后继续编辑道路中线属性。

## 输入条件

- CAD 选择对象：`DnRoadCenterlineEntity`，双击时由托管插件传入实体 handle。
- 用户输入参数：道路名称、道路等级、是否关联数模、数模 handle、桩号标注间距。
- 已有设计实体：道路中线实体。
- 外部数据：无。

## 输出结果

- CAD 图形实体：更新后的 `DnRoadCenterlineEntity`。
- 领域实体：更新后的 `RoadCenterlineProperties`。
- 表格或报告：无。
- 更新通知或重建请求：桩号标注间距变化时重建显示；仅属性变化时不重建几何。

## 操作流程

1. 用户创建道路中线完成后，系统自动打开道路中线属性窗口。
2. 用户也可双击已有道路中线实体。
3. 托管 Ribbon 插件识别 `DNROADCENTERLINEENTITY`，取得实体 handle。
4. 插件发送 `RD_ALIGN_CENTERLINE_EDIT_HANDLE`，C++ Bridge 读取实体数据并生成 WPF 请求文件。
5. WPF 窗口展示道路名称、道路等级、数模关联和桩号设置入口。
6. 用户确认后，WPF 通过回写桥接命令应用修改。

## 关键业务规则

- 道路名称默认按 `K1`、`K2`、`K3` 递增，用户可手工修改。
- 道路等级使用固定枚举列表。
- 数模关联第一版只保存被选对象 handle；WPF 不读取 ObjectARX 自定义实体类型，不执行高程采样。
- 用户只修改道路名称、道路等级或数模关联 handle 时，不触发道路中线几何重建。
- 桩号标注间距变化时需要刷新桩号显示。
- 属性窗口不得直接操作 `AcDbEntity`、`AcDbObjectId` 或 `ads_name`。

## 可复用性说明

- 可复用内容：道路中线属性 DTO、handle 编辑入口、请求/响应数据协议。
- 临时原型内容：当前 Bridge 使用请求/响应文件传递数据。
- 正式复用前需要改造的内容：替换为正式进程内 Bridge 或统一 UI 通道，并补充数模实体校验。

## 与其他模块的依赖关系

- 上游模块：道路中线创建。
- 下游模块：纵断面取高、横断面、交叉口、出图算量。
- 实体联动行为：第一版保存数模 handle，后续接入统一关系管理。

## 后续对接正式 EICAD 功能的注意事项

- 数模关联的实体校验和高程查询必须由 C++ Adapter / Service 完成。
- UI 技术栈可以替换，但业务规则和实体读写边界不变。

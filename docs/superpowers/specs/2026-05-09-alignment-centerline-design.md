# 平面布线与道路中线实体设计规格

## 目标

在 RoadProto 中新增 `ALIGNMENT` 平面设计模块，实现第一版“平面布线”原型：用户在 AutoCAD 图上依次点取路线控制点，系统按交点自动生成道路中线五单元组合，即直线、缓和曲线、圆曲线、缓和曲线、直线，并生成可持久化、可双击编辑、带桩号标注和基础夹点的道路中线自定义实体。

## 方案比较

推荐方案是“沿用地形模块模式的 C++ 核心原型”。领域层实现平曲线、缓和曲线、桩号和属性数据；`cad_adapter/objectarx` 实现点取、动态预览、自定义实体、标注绘制、夹点和临时交互；托管 Ribbon 插件创建按钮和双击转发。后续新增或重做用户对话框应按全局规则统一使用 WPF。

第二个方案是“先做 WPF 全量属性窗口和 Bridge”，界面体验更接近正式产品，但当前项目还没有 unmanaged ARX 与 WPF 的完整参数 Bridge，容易把本次重点从道路中线核心能力转移到跨语言 UI 接入。

第三个方案是“直接做完整路线设计引擎”，覆盖等级规范、超高、加宽、纵断面联动、要素表和复杂夹点实时拖拽。它更完整，但会超过 V0.1 框架阶段范围，也不利于先沉淀清晰的实体和算法边界。

## 第一版范围

第一版完成可构建、可手工验证的平面布线主链路：

- 新增 `ALIGNMENT` 模块，Ribbon 面板标题为 `平面设计`。
- 新增 `RD_ALIGN_CENTERLINE_CREATE` 命令，显示名为 `平面布线`。
- 新增 `RD_ALIGN_CURVE_PARAM_EDIT` 命令，显示名为 `编辑平曲线参数`。
- 新增内部命令 `RD_ALIGN_CENTERLINE_EDIT_HANDLE`，供双击道路中线实体时按 handle 打开属性编辑。
- 新增 `DnRoadCenterlineEntity` 自定义实体，DXF 名为 `DNROADCENTERLINEENTITY`，中文业务名称为 `道路中线自定义实体`。
- 支持用户点取起点、交点、第三点后生成第一组五单元；继续点取后追加控制点并重建整条路线。
- 默认道路名称按当前图中已有道路中线数量生成 `K1`、`K2`、`K3`。
- 实体保存道路名称、道路等级、是否关联数模、关联数模 handle、桩号标注间距、控制点、平曲线参数和计算后的路线要素。
- 默认起点桩号为 `K0+000`，按标注间距生成百米桩号，默认间距 `100m`。
- 绘制特征点桩号标注、圆曲线半径、圆曲线长度、缓和曲线长度。
- 双击实体弹出属性编辑窗口，可修改道路名称、道路等级、数模关联和桩号标注间距。
- `编辑平曲线参数` 选择道路后打开参数窗口，编辑目标曲线的 `T1`、`T2`、`LS1`、`R`、`LS2`。
- 基础夹点包括起点、终点、交点、圆曲线中点、圆弧两端点和直缓点/缓直点。第一版以拖动结束后重建实体为准，命令点取过程提供实时预览。

## 第一版不包含

第一版不实现纵断面取高、超高、加宽、完整规范表、路线要素表导出、单独可编辑的 `AcDbText` 标注实体、源地形 reactor 自动更新、跨 DWG 路线文件格式、Civil 3D 级别的多曲线约束求解器。数模关联先保存 handle 并通过关系管理器表达依赖，为后续纵断面取高预留入口。

## 缓和曲线规则

道路缓和曲线采用公路设计常用回旋线。核心关系：

```text
rho * l = A^2
A^2 = R * Ls
k(s) = s / (R * Ls)
beta(s) = s^2 / (2 * R * Ls)
beta0 = Ls / (2 * R)
```

其中 `rho` 为回旋线上某点曲率半径，`l` 为该点到回旋线起点的曲线长，`A` 为回旋线参数，`R` 为相接圆曲线半径，`Ls` 为缓和曲线长度，`k(s)` 为曲率，`beta(s)` 为切线角。实现采用领域层函数生成解析要素，并用稳定采样点驱动 CAD 显示；后续可用更高精度 Fresnel 积分替换内部采样而不影响实体接口。

参考依据：

- Autodesk Civil 3D 缓和曲线说明：回旋线常用于公路和铁路，曲率从直线端 0 线性增至圆曲线曲率。
- 平面线形教材资料：回旋线基本公式为 `rho * l = A^2`，终点满足 `R * Ls = A^2`。

## 数据模型

领域层新增 `src/domain/alignment`：

- `AlignmentPoint2d`：二维控制点。
- `RoadGrade`：高速公路、一级道路、二级道路、三级道路、四级道路、城市快速路、城市主干道、城市次干道、城市支路、其他道路、等外公路。
- `Station` 与 `StationFormatter`：路线里程与 `K0+000` 格式化。
- `RoadCenterlineProperties`：道路名称、道路等级、数模关联开关、数模 handle、桩号标注间距。
- `HorizontalCurveParameters`：`T1`、`T2`、`LS1`、`R`、`LS2`。
- `HorizontalAlignmentElement`：直线、第一缓和曲线、圆曲线、第二缓和曲线。
- `HorizontalAlignmentFeaturePoint`：起点、终点、交点、直缓点、缓圆点、圆缓点、缓直点、直圆点、圆直点、直线直线相接点、圆曲线中点。
- `HorizontalAlignmentBuilder`：由控制点和参数构建路线元素、桩号和特征点。

`T1` 和 `T2` 在几何上与 `R`、`LS1`、`LS2`、转角存在约束。第一版参数编辑器采用“主参数 + 目标切线长”的方式：`R`、`LS1`、`LS2` 是曲率主参数；`T1`、`T2` 作为可编辑布设约束。保存时 builder 会校核组合是否可解，必要时重算依赖值；无法满足时拒绝保存并提示用户减小半径、减小缓和曲线长或增大切线空间。

## 命令交互

`RD_ALIGN_CENTERLINE_CREATE` 流程：

1. 提示用户选择起点。
2. 提示用户选择交点。
3. 提示用户选择第三点，生成第一组五单元并显示预览。
4. 用户继续点取后，追加控制点，前一个末端点转为新的内部交点，路线整体重建。
5. 用户按 `Enter`、鼠标右键或 `ESC` 结束点取，打开道路中线属性窗口。
6. 用户确认后插入 `DnRoadCenterlineEntity`；取消则删除预览。

`RD_ALIGN_CURVE_PARAM_EDIT` 流程：

1. 优先识别当前预选的 `DnRoadCenterlineEntity`，否则提示选择道路中线。
2. 如果实体包含多组平曲线，按用户点击位置选择最近曲线；没有点击位置时默认编辑第一组曲线。
3. 打开平曲线参数窗口，显示并允许编辑 `T1`、`T2`、`LS1`、`R`、`LS2`。
4. 参数通过领域 builder 校核后写回实体并刷新图形。

双击流程：

1. 托管 Ribbon 插件监听 AutoCAD `BeginDoubleClick`。
2. 识别 `DNROADCENTERLINEENTITY` 后发送 `RD_ALIGN_CENTERLINE_EDIT_HANDLE <handle>`。
3. C++ ObjectARX 打开实体，弹出道路中线属性窗口。

## UI 与 Ribbon

C++ 模块层负责向 `RibbonModel` 注册 `ALIGNMENT` 面板和命令元数据。AutoCAD 可见 Ribbon 继续由 `src/ui/wpf/RoadProto.Terrain.UI/AutoCad/RoadProtoRibbonExtension.cs` 创建，因为当前托管插件已经负责 RoadProto 选项卡、按钮尺寸、图标和双击事件。为避免以后遗漏，本次同步更新 `docs/rules/command_and_module_rules.md`，明确新增 Ribbon 命令时必须同时更新 C++ Ribbon 元数据、托管可见 Ribbon、图标策略和模块文档。

Ribbon 按钮采用与地形模块一致的 `RibbonItemSize.Standard`、横向小按钮和内置矢量图标。新增图标使用道路中线图形语义，例如蓝色中心线、灰色切线、黄色曲线节点。

## 自定义实体绘制

`DnRoadCenterlineEntity` 在 `subWorldDraw` 中绘制：

- 道路中线：按直线、缓和曲线和圆曲线采样后的连续线。
- 特征点：起点、终点、交点、直缓点、缓圆点、圆缓点、缓直点、直圆点、圆直点、直线直线相接点。
- 桩号标注：默认每 `100m` 绘制一次，格式为 `K0+000`、`K0+100`。
- 参数标注：圆曲线半径 `R`、圆曲线长度 `Lc`、第一和第二缓和曲线长度 `LS1`、`LS2`。

第一版标注由实体自身绘制，不额外生成独立 `AcDbText`。这样属性、夹点和参数变化后标注能随实体同步刷新；后续若需要出图级文本，再增加“生成独立标注”命令。

## 数模关联

道路中线属性窗口提供 `是否关联数模` checkbox。勾选后可点击 `选择数模`，临时隐藏窗口并让用户选择图中的数模对象。第一版只保存被选对象 handle，WPF 不读取 ObjectARX 自定义实体类型；后续由 C++ Adapter/Service 校验实体并通过 `EntityRelationManager` 表达道路中线对地形数模的依赖。第一版不从数模取高，只保证关联信息可保存、可编辑、可为后续纵断面取高使用。

## 夹点行为

第一版夹点规则：

- 起点、终点：移动对应控制点并重建整条路线。
- 交点：移动对应 PI 点并重建相邻平曲线。
- 圆曲线中点：按圆心方向调整半径 `R`，保留缓和曲线长度，重建该组平曲线。
- 圆弧两端点：投影到对应曲线方向，调整圆曲线长度和相邻缓和曲线连接位置，重建参数。
- 直缓点、缓直点：按交点到切线点距离比例缩放该侧平曲线包络，影响 `T1/T2` 和 `LS1/LS2`。

拖动过程中的连续橡皮筋反馈作为后续增强；第一版确保拖动完成后的实体几何、桩号和标注一致刷新。

## 测试策略

核心测试不依赖 AutoCAD：

- 回旋线公式：`A^2 = R * Ls`、曲率线性变化、终点切线角。
- 三点生成五单元：元素顺序为直线、缓和曲线、圆曲线、缓和曲线、直线。
- 桩号格式：`0` 为 `K0+000`，`100` 为 `K0+100`，`1234.56` 为 `K1+234.560`。
- 特征点桩号：起点、直缓点、缓圆点、圆缓点、缓直点、终点桩号单调递增。
- 参数校核：无效半径、负缓和曲线长、切线长度不足时返回明确错误。
- 模块和 Ribbon 元数据：`ALIGNMENT` 模块注册两个可见命令和一个内部 handle 命令。

AutoCAD 手工验证：

- 加载 ARX 和托管 Ribbon 后出现平面设计按钮。
- 点取三点生成道路中线实体。
- 继续点取可追加路线。
- 百米桩号、特征点桩号、半径和缓和曲线长度可见。
- 双击实体可编辑属性。
- `编辑平曲线参数` 可修改曲线参数并刷新图形。
- 保存 DWG 后重新打开并 `REGEN` 不崩溃。
- 基础夹点拖动后实体和标注刷新。

## 风险与处理

平曲线参数存在几何约束，不能把 `T1`、`T2`、`LS1`、`R`、`LS2` 当成永远独立的自由变量。第一版通过领域 builder 集中校核并拒绝无解组合，避免把错误几何写入实体。

夹点拖动实时预览在 ObjectARX 中工作量较大，第一版先保证 grip 结束后的结果正确；命令创建过程先提供实时预览，后续再增加 grip 拖动过程的连续显示。

道路等级第一版只作为属性保存和 UI 下拉，不驱动半径、缓和曲线最小值和设计速度规范。后续可把等级和设计速度接入参数推荐器。

## 规格自检

- 本规格没有未决内容。
- 本规格将 CAD/ObjectARX 能力限制在 `cad_adapter/objectarx` 和托管 Ribbon 插件内。
- 本规格将平曲线、桩号和参数校核放在不依赖 ObjectARX 的领域层。
- 本规格明确第一版范围和暂不包含内容，适合作为单次实施计划输入。

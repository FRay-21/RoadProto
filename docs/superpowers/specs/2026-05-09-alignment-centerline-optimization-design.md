# 平面布线交互与 WPF 参数窗口优化设计

## 目标

优化 `ALIGNMENT` 平面布线原型，使道路中线创建、编辑、显示、标注和夹点行为更接近道路设计工作流。

## 当前阶段 UI 规则

当前 RoadProto 原型阶段，用户参数窗口、编辑窗口、设置窗口和二级配置窗口统一先使用 WPF 实现。WPF 只负责参数展示、输入和命令转发，不直接操作 `AcDbEntity`、`AcDbObjectId`、`ads_name` 等 ObjectARX 类型。C++ ObjectARX 仍负责点取、实体访问、实时预览、几何计算、绘制、持久化和夹点。

未来如果整体 UI 技术栈替换为 MFC、Qt、AutoCAD Palette、其他 UI 或 EICAD 正式 UI，也必须通过 Bridge / Adapter 调用 C++ 核心能力，不能移动几何算法、自定义实体和 DWG 持久化逻辑。

## 功能设计

1. `RD_ALIGN_CENTERLINE_CREATE` 点取过程中显示可见反馈。第三点后立即显示可更新的道路中线预览实体，继续点取控制点时实时重建预览；后续点取阶段不显示 AutoCAD 自带橡皮筋线或旧实体末端短直线。
2. 道路中线按元素类型着色：直线段青色，第一缓和曲线红色，圆曲线黄色，第二缓和曲线洋红色。
3. 属性、平曲线参数和桩号标注设置都使用 WPF 窗口。C++ 命令写入 DTO 请求文件并排队调用托管命令；WPF 写回 DTO 响应文件后发送内部 C++ 命令应用到实体。
4. 双击道路中线打开同一个 WPF 道路中线窗口，可修改道路名称、道路等级、数模关联、桩号间距，并按“第 x 个平曲线”翻页编辑每个交点对应的平曲线参数。
5. 平曲线几何按道路设计主点公式构建。回旋线满足 `A^2 = R * Ls`，交点为前后切线延长线交点；直缓、缓圆、圆缓、缓直和圆曲线中点按主点计算。
6. 夹点支持第一版参数拖动：控制点拖动改控制线；直缓/缓直拖动改前后切线长；缓圆/圆缓拖动改第一/第二缓和曲线长；圆曲线中点拖动改半径。拖动结束后重建实体。
7. 桩号标注改为引线标注形式：从桩号点绘制短引线到文字位置。

## 验证

- 核心测试覆盖回旋线 `m/p/T` 公式、交点主点位置、五单元和多交点构建。
- 托管 WPF 工程 Debug/Release 构建通过。
- ARX Debug/Release 构建通过。
- AutoCAD 图形界面需人工点验 Ribbon、WPF 弹窗、实时预览、双击编辑和夹点拖动。

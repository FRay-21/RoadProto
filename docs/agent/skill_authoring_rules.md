# Agent Skill 写作规则

## 用途

RoadProto Agent skill 是给本地 Agent 后端读取的 markdown 能力说明。它帮助模型判断何时调用工具、如何补参数、什么时候追问、什么时候拒绝自动执行。

当前架构下，skill 同时承担两类职责：

- 作为模型 prompt 上下文，帮助模型回答普通问答、解释软件操作和理解工具语义。
- 作为开发契约，指导 `AgentPlanner` 和具体工具 planner 把自然语言解析为结构化 `toolCall`。

因此，只改 skill 不一定改变真实 CAD 工具调用。凡是希望“用户怎么说都能生成工具卡片”的规则，必须同时补到对应 planner 代码和自动化测试中。

## 文件位置

工具 skill 放在：

```text
docs/agent/skills/<module>/<tool_name>.md
```

## 必备内容

每份 skill 必须包含：

- 工具 ID。
- 所属模块。
- 对应业务文档。
- 触发语义。
- 触发词、别名词和不触发场景。
- 必要参数。
- 可选参数。
- 默认值规则。
- 局部修改规则。
- 缺参追问规则。
- 执行前确认文案。
- 禁止事项。
- 示例用户表达。
- 示例工具 JSON。
- 示例用户表达到结构化计划的映射。

## 写作原则

- 用明确规则，不写开放式猜测。
- 缺少工程参数时优先追问。
- 只能引用已存在或用户明确提供的 CAD handle。
- 不允许凭空绑定实体。
- 默认值规则必须说明来源，例如来自道路等级默认值、同侧同类型部件推断或用户显式输入。
- 局部修改优先表达为受控增量操作，例如 `componentOperations`，不要为了一个局部修改虚构完整部件列表。
- 示例必须覆盖同义说法、缺省参数说法和至少一个不应触发工具的说法。
- 新增或调整 skill 规则时，应同步增加 `AgentPlanner` 测试，验证自然语言能生成预期 `toolCall`。

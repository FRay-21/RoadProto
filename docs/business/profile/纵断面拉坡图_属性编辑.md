# 纵断面拉坡图属性编辑

## 功能范围

本功能用于编辑已创建的 `DnProfileGradeGraphEntity` 纵断面拉坡图属性。用户双击拉坡图实体后，由托管 Ribbon 插件识别实体 handle，并通过 C++ Bridge 打开 WPF 属性窗口。

## 命令

- 内部 handle 编辑命令：`RD_PROFILE_GRADE_GRAPH_EDIT_HANDLE`
- 内部对话框结果应用命令：`RD_PROFILE_APPLY_DIALOG_FILE`
- 托管窗口命令：`RD_PROFILE_SHOW_WPF_DIALOG`
- 模块编码：`PROFILE`
- Ribbon 位置：内部桥接命令不挂接 Ribbon；用户创建入口位于 `RoadProto / 纵断面设计 / 纵断面拉坡图`

## 属性窗口

窗口当前包含：

- 拉坡图名称。
- 地面线颜色，默认青绿色 ACI 4，支持常用 ACI 颜色选择。
- 地面线宽度，默认 `1`，当前有效范围为 `(0, 2.11]mm`。
- 地面线精度，默认 `10`；数模来源修改后会按新精度重新采样，DMX 来源保留文件点列。
- 纵向比例，下拉支持 `1:1`、`1:10`、`1:100`。
- 网格线间距，默认 `10`。
- 数据来源、道路名称、道路 handle、数模 handle、DMX 文件路径和样本数量等只读信息。
- `更新地面线` 按钮。

用户点击“确定”时只回写属性并刷新实体显示；点击“取消”时不修改实体。

## 更新地面线

实体来源为 DMX 文件时：

1. 实体保存创建时选择的 DMX 文件路径。
2. 用户点击“更新地面线”。
3. C++ 回写命令重新读取该路径。
4. 读取成功且布局校验通过后，替换实体内的地面线样本并刷新显示。
5. 读取失败时保留原实体数据，并在命令行输出错误。

实体来源为关联数模时：

- WPF 中“更新地面线”按钮置灰。
- C++ 回写命令收到数模来源更新请求时也会忽略并提示。
- 用户修改地面线精度并点击“确定”时，会按保存的道路中线 handle 和数模 handle 重新采样一次。
- 后续路线移动、数模更新和拉坡图重建应接入统一实体关系与联动更新机制。

## 当前限制

- 属性窗口通过临时请求/响应文件与 C++ ObjectARX 通信，后续可替换为更正式的进程内 Bridge。
- 当前仅支持编辑拉坡图基础显示属性和 DMX 文件更新；竖曲线设计、设计线、坡度坡长表和标注扩展留待后续功能。
- 完整鼠标双击弹窗和窗口交互仍需要在 AutoCAD 图形界面中人工点验。

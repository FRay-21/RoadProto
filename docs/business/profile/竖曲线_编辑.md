# 竖曲线编辑

> 文档粒度要求：本文只描述竖曲线编辑和 WPF 回写桥接，不描述创建、夹点新增或右键删除流程。

## 基本信息

- 功能名称：竖曲线编辑
- 所属模块：纵断面设计
- 命令名称：`RD_PROFILE_VERTICAL_CURVE_EDIT_HANDLE`、`RD_PROFILE_VERTICAL_CURVE_APPLY_DIALOG_FILE`
- 对应代码入口：`src/modules/profile/ProfileModule.cpp`
- 业务文档维护人：RoadProto
- 原型版本：V0.1 ProfileGradeGraph
- 是否可复用：部分可复用

## 功能背景

竖曲线创建后，需要通过双击、handle 桥接或 WPF 参数窗口进行属性编辑。当前 Task 4 只登记内部命令和文档路径，后续任务再实现实体读取、窗口展示和回写。

## 业务目标

为竖曲线编辑提供两个内部桥接命令：一个按 handle 定位待编辑对象，一个应用 WPF 对话框响应文件。两者共同支撑 UI 可替换的编辑流程。

## 适用场景

适用于已经创建竖曲线实体的 DWG，用于后续修改竖曲线参数、PVI 关系或显示属性。

## 输入条件

- CAD 选择对象：后续通过 handle 定位竖曲线实体。
- 用户输入参数：后续由 WPF 编辑窗口以 DTO 或响应文件形式提供。
- 已有设计实体：竖曲线实体及其所属纵断面拉坡图。
- 外部数据：WPF 对话框响应文件。

## 输出结果

- CAD 图形实体：后续更新竖曲线实体显示和持久化数据。
- 领域实体：后续更新竖曲线参数模型。
- 表格或报告：无。
- 更新通知或重建请求：后续触发相关显示对象刷新。

## 操作流程

1. 双击或桥接流程发送 `RD_PROFILE_VERTICAL_CURVE_EDIT_HANDLE` 并传入实体 handle。
2. C++ 侧读取实体数据并准备编辑 DTO。
3. WPF 只负责展示 DTO、收集参数和写出响应。
4. `RD_PROFILE_VERTICAL_CURVE_APPLY_DIALOG_FILE` 读取响应文件并由 C++ application/domain/cad_adapter 完成校验、计算和实体修改。

## 关键业务规则

- WPF 不直接操作 `AcDbEntity`、`AcDbObjectId` 或 `ads_name`。
- WPF 只负责 DTO/参数输入，不承载竖曲线核心计算。
- handle 命令和响应文件命令均为内部桥接命令，默认不挂接 Ribbon。
- 实体修改必须通过 C++ application/domain/cad_adapter 完成。

## 可复用性说明

- 可复用内容：handle 编辑入口、对话框响应文件桥接模式、竖曲线参数校验流程。
- 临时原型内容：当前命令过程为空实现，只用于框架接线。
- 正式复用前需要改造的内容：补充 DTO、文件格式、错误提示、事务封装和实体刷新。

## 与其他模块的依赖关系

- 上游模块：纵断面拉坡图、竖曲线创建。
- 下游模块：夹点编辑、右键编辑、出图和算量。
- 实体联动行为：编辑后应通过统一关系管理机制通知依赖对象。

## 后续对接正式 EICAD 功能的注意事项

- 对话框数据结构应保持 UI 技术栈无关。
- 参数校验和曲线计算不应写入 WPF 事件。

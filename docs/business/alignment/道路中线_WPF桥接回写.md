# 道路中线 / WPF 桥接回写

## 基本信息

- 功能名称：道路中线 WPF 桥接回写
- 所属模块：平面设计
- 命令名称：`RD_ALIGN_APPLY_DIALOG_FILE`
- 对应代码入口：`AlignmentDialogBridge.*`、`AlignmentDialogCommands.cs`
- 业务文档维护人：RoadProto
- 原型版本：v0.1.8
- 是否可复用：部分可复用

## 功能背景

当前原型阶段道路中线窗口在托管 WPF 插件中实现，C++ ObjectARX 核心通过请求/响应文件与 WPF 交换数据。本功能承接 WPF 确认后的回写流程。

## 业务目标

- 让 WPF 窗口不直接访问 ObjectARX 类型。
- C++ 侧统一读取 WPF 响应文件并更新道路中线实体。
- 根据响应内容判断是否需要重建几何或仅更新属性。

## 适用场景

- 道路中线属性窗口确认。
- 平曲线参数页确认。
- 桩号标注设置二级窗口确认。

## 输入条件

- CAD 选择对象：响应文件中携带目标实体 handle。
- 用户输入参数：WPF 响应文件中的道路属性、桩号间距和平曲线参数。
- 已有设计实体：道路中线实体。
- 外部数据：Bridge 请求/响应临时文件。

## 输出结果

- CAD 图形实体：更新后的道路中线实体。
- 领域实体：更新后的属性和参数集合。
- 表格或报告：无。
- 更新通知或重建请求：根据响应内容决定是否重建当前实体。

## 操作流程

1. C++ Bridge 为目标实体生成 WPF 请求文件。
2. 托管命令 `RD_ALIGN_SHOW_WPF_DIALOG` 打开 WPF 道路中线窗口。
3. 用户确认后，WPF 写出响应文件。
4. WPF 发送 `RD_ALIGN_APPLY_DIALOG_FILE` 并传入响应文件路径。
5. C++ 读取响应文件，校验 handle 和参数。
6. C++ 更新实体属性，必要时按领域层规则重建道路中线。

## 关键业务规则

- 响应文件只传递 DTO 数据，不传递 ObjectARX 指针。
- 只修改道路名称、道路等级或数模 handle 时，不重建几何。
- 修改桩号间距或平曲线参数时，需要重建显示数据。
- 回写失败必须通过编辑器输出明确错误信息，不得静默失败。
- Bridge 层只做数据转换和调用转发，不写平曲线算法。

## 可复用性说明

- 可复用内容：请求/响应 DTO、WPF 与 C++ 核心解耦边界、handle 定位实体的回写流程。
- 临时原型内容：文件式 Bridge。
- 正式复用前需要改造的内容：替换为进程内 Bridge、统一错误码和事务封装。

## 与其他模块的依赖关系

- 上游模块：道路中线属性编辑、平曲线参数编辑。
- 下游模块：道路中线实体显示和 DWG 持久化。
- 实体联动行为：第一版只更新当前实体，后续再接联动通知。

## 后续对接正式 EICAD 功能的注意事项

- UI 技术栈替换时仍必须保留 Bridge / Adapter 解耦。
- 响应文件格式变更时需要同步核心测试或集成测试说明。

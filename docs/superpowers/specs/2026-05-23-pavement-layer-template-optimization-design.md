# 路面结构层模板显示与编辑优化设计

## 背景

现有路面结构层模板已经具备独立 DWG 实体、WPF 参数窗口、每层 RGB 颜色、四边形/梯形预览几何、`.rpavement.xml` 导入导出，以及道路模型读取模板生成结构层显示的基础能力。

本次优化聚焦两个问题：

- 预览和 DWG 模板实体标注不够专业，缩放后的标注比例和表达方式不符合 CAD 制图习惯。
- 右侧参数区一次展开所有结构层，层数较多时滚动成本过高。

设计目标是在不改变道路模型结构层生成口径的前提下，优化模板编辑体验和模板实体显示表达。WPF 仍只负责展示、交互和 XML；字段归一化、模板数据和实体持久化仍沉淀在 C++ 领域层和 ObjectARX 实体中。

## 目标

- 优化 WPF 预览图和 `DnPavementLayerTemplateEntity` 的标注样式，让层名/厚度、加宽尺寸线和坡度标注更接近 CAD 制图习惯。
- 让预览标注随模型比例保持固定图纸尺寸，不再保持屏幕恒定尺寸导致缩放后字体视觉比例漂移。
- 整体缩小标注尺寸。
- 将结构层类型/名称和厚度放在同一行显示。
- 加宽不再使用“内侧加宽/外侧加宽”文字，改为仅在实际加宽位置绘制 CAD 风格尺寸标注。
- 坡度标注只显示 `1:1`、`1:n`，放在对应侧边中心附近。
- 右侧参数区改为“通用参数 + 当前选中层参数”，支持预览图点选层、层号输入和上/下按钮切换当前层。
- RGB 颜色预览块可点击，弹出索引色选择对话框，回填对应 RGB。
- 每层新增填充类型字段，通用参数新增显示方式字段。
- WPF 预览和 `DnPavementLayerTemplateEntity` 按显示方式渲染；道路模型中的路面结构层保持现有仅颜色显示。

## 非目标

- 不新增材料库、规范库或结构组合库。
- 不改变道路模型结构层几何生成规则。
- 不改变道路模型中的结构层显示策略，道路模型仍只按颜色显示。
- 不把 CAD 业务逻辑写入 WPF。
- 不引入可独立选择、编辑或算量的真实结构层面实体。

## 数据模型

在 `PavementLayerTemplateLayer` 中新增：

- `hatchPattern`：结构层填充类型，字符串保存 AutoCAD 预定义填充名，默认 `SOLID`。

在 `PavementLayerTemplateProperties` 中新增：

- `displayMode`：结构层模板显示方式。

显示方式固定三种：

- `Color`：按颜色显示，保持当前颜色弱化填充模式。
- `Hatch`：按填充显示，使用填充线型表达，颜色使用中性浅色或层边线色的低强调版本。
- `HatchAndColor`：填充和颜色同时显示，填充线型叠加在层颜色弱化填充上。

填充下拉框使用 V0.1 的固定预定义列表，不做完整 AutoCAD 填充库扫描：

- `SOLID`
- `ANSI31`
- `ANSI32`
- `ANSI33`
- `ANSI34`
- `ANSI35`
- `ANSI36`
- `ANSI37`
- `ANSI38`
- `CROSS`
- `DOTS`
- `GRAVEL`
- `EARTH`

领域层负责默认值、合法值归一化和编码往返。WPF、桥接文件、XML 和 DWG 持久化都读写同一字段。

## WPF 编辑交互

右侧参数区改为两段：

- 通用参数：预览宽度、结构层数量、显示方式。
- 当前层参数：当前编辑层号、上/下切换按钮、类型、名称、RGB、索引色选择、填充类型、厚度、加宽和坡度参数。

层号交互规则：

- 当前层号按 1 基显示。
- 用户可输入层号；输入越界时夹到 `1..层数`。
- 预览图点击某层后，右侧层号和参数切换到该层。
- 上按钮让当前层号减 1，例如 5 变 4；下按钮让当前层号加 1。
- 按钮使用图标而不是纯文字，并提供 tooltip。

颜色交互规则：

- RGB 文本框继续保留。
- 颜色预览块显示当前 RGB。
- 点击颜色预览块弹出索引色选择对话框。
- 索引色选择使用固定 ACI 常用色板，用户选中后回填 RGB 文本框和预览块。

## WPF 预览显示

预览继续使用当前四边形/梯形几何规则。

显示方式：

- `Color`：绘制弱化层颜色填充和层色边线。
- `Hatch`：绘制填充线型和边线，不显示色块填充。
- `HatchAndColor`：先绘制弱化层颜色填充，再绘制填充线型，最后绘制边线和标注。

标注规则：

- 层名和厚度同一行，格式为 `基层  厚 0.18` 或 `基层  厚 0.16/0.20`。
- 标注字号小于现状，按图纸/世界比例转换到屏幕，随预览缩放变化，不做屏幕固定大小文字。
- 加宽仅在实际加宽值非零的一侧显示。
- 加宽显示为 CAD 风格尺寸线：两条界线、尺寸线、箭头或三角箭头，以及数值。
- 加宽尺寸线只标注当前层相对上一层底边产生的加宽段，不标注整层宽度。
- 坡度仅在实际坡度非零的一侧显示。
- 坡度文字为 `1:1`、`1:n` 或 `1:-n`，放在对应侧边中心旁边。
- 标注整体应避开层内主文字，避免全部挤在结构层中心。

图层点选：

- 每层预览 polygon 参与点击命中。
- 点击命中层后更新当前编辑层并重绘选中态。
- 选中层边线加粗或增加高亮描边，但不改变模板保存颜色。

## DWG 模板实体显示

`DnPavementLayerTemplateEntity` 与 WPF 使用同一份字段和同一套显示语义。

标注规则与 WPF 保持一致：

- 层名/厚度一行。
- 加宽使用尺寸线，不用说明文字。
- 坡度显示 `1:n`，放在侧边中心附近。
- 标注尺寸整体小于当前版本。
- 标注高度按模板显示比例得到固定图纸尺寸；用户缩放 CAD 视口时标注自然随实体缩放。

显示方式：

- `Color`：沿用当前预混合弱化填充色和层 RGB 边线。
- `Hatch`：绘制 hatch 风格填充线或 ObjectARX hatch 表达，并保留边线。
- `HatchAndColor`：弱化颜色填充与 hatch 叠加。

实现上优先保持在 `DnPavementLayerTemplateEntity` 的自绘表达中，避免生成额外可选中实体。若 ObjectARX hatch API 在 `AcGiWorldDraw` 下限制过多，可用轻量自绘 hatch 线近似表达，字段仍保存 AutoCAD 预定义填充名。

## 桥接与 XML

WPF 请求/响应文件新增字段：

- `displayMode`
- `layer.N.hatchPattern`

XML 新增属性：

- `displayMode`
- `hatchPattern`

兼容规则：

- 旧请求/旧 XML 缺少 `displayMode` 时默认为 `Color`。
- 旧请求/旧 XML 缺少 `hatchPattern` 时每层默认为 `SOLID`。
- 旧 DWG 数据缺少新增字段时也按上述默认值补齐。

## 道路模型显示边界

本次新增显示方式只作用于：

- WPF 路面结构层模板预览。
- `DnPavementLayerTemplateEntity` 自定义实体。

不作用于：

- 横断面戴帽生成的 `DnRoadModelEntity` 结构层显示。
- 查看横断面中来自道路模型的结构层预览。

道路模型结构层继续使用当前每层 RGB 颜色和弱化填充表达，避免模板编辑显示模式影响已经生成的道路模型视觉和性能。

## 测试策略

核心测试：

- 领域层默认 `displayMode=Color`、默认每层 `hatchPattern=SOLID`。
- 合法填充名和显示方式编码往返；非法值归一为默认值。
- DWG 实体持久化源码契约包含新增字段读写和显示模式分支。
- DWG 实体标注源码契约包含层名/厚度同行、加宽尺寸线和坡度 `1:n`。

托管测试：

- 请求/响应文件读写新增字段并兼容缺省值。
- XML 读写新增字段并兼容缺省值。
- WPF 窗口源码契约包含当前层号、上/下图标按钮、预览点击选层、颜色预览点击、索引色对话框、填充下拉和显示方式下拉。
- WPF 预览源码契约包含 hatch 渲染分支、选中层高亮和加宽尺寸线。

构建验证：

- Debug/Release 核心测试。
- 托管 bridge 测试。
- WPF Release 构建。
- ARX Release 构建。

人工验证：

- 创建路面结构层模板，点击预览层后右侧只显示当前层参数。
- 输入层号和上/下按钮能稳定切层。
- 点击颜色预览块能选择索引色并回填 RGB。
- 切换显示方式后，预览和 DWG 模板实体显示对应变化。
- 道路模型生成后结构层显示仍按颜色表达，不受模板显示方式影响。

# 纵断面拉坡图创建

## 功能范围

本功能属于纵断面设计模块，用于根据道路中线和地面线数据创建纵断面拉坡图自定义实体。

当前实现覆盖：

- 选择道路中线。
- 优先从道路中线关联的地形数模采样地面高程。
- 没有关联数模或数模不可用时，选择 `.dmx` 纵地面线文件读取桩号和地面高程。
- 点取图纸插入位置。
- 创建 `DnProfileGradeGraphEntity`，绘制拉坡图名称、表头、网格线和地面线折线。

## 命令

- 用户命令：`RD_PROFILE_GRADE_GRAPH_CREATE`
- 显示名称：纵断面拉坡图
- 模块编码：`PROFILE`
- Ribbon 位置：`RoadProto / 纵断面设计 / 纵断面拉坡图`
- 业务文档：`docs/business/profile/纵断面拉坡图_创建.md`

## 数据来源

### 数模来源

当道路中线保存了有效数模 handle，且该 handle 指向 `DnTerrainTinEntity` 时，命令沿道路中线采样点调用数模高程查询，生成 `桩号-高程` 数据。

当前采样精度使用实体属性中的默认精度 `10`，表示沿道路中线按桩号间距重新插值取样后查询数模高程。数模来源的拉坡图在属性窗口中不允许通过“更新地面线”按钮直接更新；用户修改精度并确定时，会按保存的道路中线和数模 handle 重新采样一次。后续路线和数模联动更新将通过统一关系管理机制扩展。

### DMX 来源

当道路中线没有可用数模时，命令弹出文件选择，让用户选择 `.dmx` 纵地面线文件。文件格式为每行一组：

```text
ZH H
```

- `ZH`：桩号，支持普通数值，也支持断链写法如 `37123.456_2`。当前原型读取断链写法中的桩号数值部分。
- `H`：地面高程。
- 空行和以 `//` 开头的注释行会跳过。
- 同桩号的连续记录会保留，用于表达局部竖向突变或原始数据重复。
- 无效行、非有限数值或少于两个有效点会被拒绝。

实体会保存 DMX 文件路径，后续属性窗口里的“更新地面线”会重新读取该路径。

## 图面表达

- X 正方向为桩号递增方向，水平比例固定按桩号 `1:1` 映射。
- Y 正方向为高程方向，纵向比例由属性选择，当前支持 `1:1`、`1:10`、`1:100`。
- 默认地面线颜色为青绿色 ACI 4。
- 默认地面线宽度为 `1`，当前有效范围为 `(0, 2.11]mm`。
- 默认地面线精度为 `10`，用于数模来源沿道路中线采样；DMX 来源保留文件中的原始点列。
- 默认网格线间距为 `10`。
- 拉坡图名称默认使用“道路名称 + 拉坡图”，道路名称为空时使用“道路拉坡图”。

## 代码位置

| 层 | 文件 | 职责 |
| --- | --- | --- |
| domain | `src/domain/profile/ProfileDmxFile.*` | 读取和校验 DMX 纵地面线文件 |
| domain | `src/domain/profile/ProfileGradeGraphModel.h` | 拉坡图来源、样本、属性和数据模型 |
| domain | `src/domain/profile/ProfileGradeGraphLayout.*` | 计算网格范围、基准高程和图面坐标映射 |
| application | `src/application/profile/ProfileGradeGraphCreateService.*` | 从准备好的样本和道路信息生成拉坡图数据 |
| cad_adapter | `src/cad_adapter/objectarx/profile/ObjectArxProfileGradeGraphCommand.*` | CAD 选择、文件对话框、数模采样和实体插入 |
| cad_adapter | `src/cad_adapter/objectarx/profile/DnProfileGradeGraphEntity.*` | 自定义实体绘制、DWG 持久化、几何范围和变换 |

## 当前限制

- 数模来源的自动更新入口已按 UI 置灰，尚未接入路线移动后的统一联动重建。
- `.dmx` 断链号当前只作为兼容输入保留，布局计算使用桩号数值部分。
- 真实 Ribbon 点击、图形界面文件对话框、双击窗口和 DWG 保存重开仍需要在完整 AutoCAD 图形界面中人工点验。

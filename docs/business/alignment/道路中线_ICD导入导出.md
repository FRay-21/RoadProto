# 道路中线 / ICD 导入导出

## 基本信息

- 功能名称：道路中线 ICD 导入导出
- 所属模块：平面设计
- 命令名称：`RD_ALIGN_CENTERLINE_EXPORT_ICD`、`RD_ALIGN_CENTERLINE_IMPORT_ICD`
- 对应代码入口：`IcdAlignmentFile.*`、`ObjectArxAlignmentCommand.cpp`
- 业务文档维护人：RoadProto
- 原型版本：v0.1.8
- 是否可复用：是

## 功能背景

ICD 是积木法线形单元文件，常用于保存立交匝道平面设计参数和程序内部线位数据交换。本功能让 RoadProto 的道路中线实体可以与 `.icd` 文件互相流转。

## 业务目标

- 将选中的道路中线实体导出为 `.icd` 文件。
- 从 `.icd` 文件导入并生成新的 `DnRoadCenterlineEntity`。
- 支持 ICD `1/2/3/4/5/6` 单元。
- 支持 ICD 工程坐标与 CAD 坐标之间转换。

## 适用场景

- 与外部道路设计程序交换路线线形。
- 保存或恢复匝道平面线形参数。
- 验证不完整缓和曲线和元素链在 DWG 内的显示与保存。

## 输入条件

- CAD 选择对象：导出时选择 `DnRoadCenterlineEntity`；导入时无需选择实体。
- 用户输入参数：导出文件路径或导入文件路径。
- 已有设计实体：导出时需要已有道路中线实体。
- 外部数据：`.icd` 文本文件。

## 输出结果

- CAD 图形实体：导入时新增 `DnRoadCenterlineEntity`。
- 领域实体：`IcdAlignmentDocument`、`HorizontalAlignment`、元素链道路中线。
- 外部文件：导出时生成 `.icd` 文件。
- 更新通知或重建请求：无。

## 操作流程

### 导出

1. 用户运行 `RD_ALIGN_CENTERLINE_EXPORT_ICD` 或点击 Ribbon `导出中线 ICD`。
2. 用户选择道路中线实体。
3. 系统读取实体内的 `HorizontalAlignment`。
4. `IcdAlignmentFile` 将道路中线转换为 ICD 文档。
5. 用户选择保存路径，系统写出 `.icd` 文件。

### 导入

1. 用户运行 `RD_ALIGN_CENTERLINE_IMPORT_ICD` 或点击 Ribbon `导入中线 ICD`。
2. 用户选择 `.icd` 文件。
3. `IcdAlignmentFile` 读取并校验 ICD 文档。
4. 系统将 ICD 工程坐标转换为 CAD 坐标，将 ICD 方位角转换为 CAD 方位角。
5. 系统按 ICD 单元构建道路中线，插入新的 `DnRoadCenterlineEntity`。

## 关键业务规则

- ICD 文件支持逗号或空白分隔。
- ICD 文件支持 `//` 行尾注释。
- ICD 结束符为 `0,0,0`。
- 起点桩号支持普通数值，也支持带断链后缀的文本，例如 `37123.456_2`。
- ICD 起点坐标中的 `x/y` 按工程坐标解释，导入时映射为 CAD `x = 工程 y`、`y = 工程 x`；导出时反向转换。
- ICD 起点方位角按工程坐标口径解释，导入时换算为 CAD 方位角，导出时反向换算。
- ICD 转向约定：`1` 为右转，`-1` 为左转；进入领域层时转换为带符号曲率。
- ICD `3` 完整缓和曲线长度为 `A^2 / EndR`。
- ICD `4` 完整缓和曲线长度为 `A^2 / StartR`。
- ICD `5/6` 不完整缓和曲线长度为 `abs(A^2 / Re - A^2 / Rs)`。
- 导入的元素链实体第一版支持显示、桩号、DWG 持久化和再次导出，暂不支持夹点编辑和 WPF 参数编辑。

## 可复用性说明

- 可复用内容：ICD 文本解析、ICD 写出、坐标系转换、ICD 单元与元素链互转、不完整缓和曲线长度换算。
- 临时原型内容：ObjectARX 文件路径选择和导入实体默认道路属性。
- 正式复用前需要改造的内容：文件编码策略、错误定位信息、更多 ICD 单元兼容性和导入后的交互编辑。

## 与其他模块的依赖关系

- 上游模块：道路中线创建、平曲线参数编辑、元素链能力。
- 下游模块：纵断面、横断面、交叉口、外部路线数据交换。
- 实体联动行为：导入导出不触发现有设计实体重建。

## 后续对接正式 EICAD 功能的注意事项

- ICD 文件格式规则应继续保持在领域层，不写入 ObjectARX 命令回调。
- 导入后的元素链编辑能力应复用元素链模型，而不是转换回五单元控制点模型。

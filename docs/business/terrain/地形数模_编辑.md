# 地形数模 / 编辑

## 基本信息

- 功能名称：地形数模编辑
- 所属模块：地形数模
- 命令名称：`DN_TERRAIN_TIN_EDIT`、`DN_TERRAIN_TIN_EDIT_HANDLE`
- 对应代码入口：`ObjectArxTerrainTinCommand.cpp`、`DnTerrainTinEntity.*`
- 业务文档维护人：RoadProto
- 原型版本：v0.1.6
- 是否可复用：部分可复用

## 功能背景

地形构网完成后，用户需要调整显示模式、最大三角边长和退化三角形过滤，并可使用实体内保存的点重新生成 TIN。

## 业务目标

- 支持选择已有 `DnTerrainTinEntity` 并打开编辑流程。
- 支持双击数模实体后按 handle 打开同一编辑流程。
- 支持修改显示模式和构网参数后重新生成数模。

## 适用场景

- 已有地形数模显示过密或边界不符合预期。
- 构网参数需要二次调整。
- DWG 保存重开后继续编辑数模。

## 输入条件

- CAD 选择对象：`DnTerrainTinEntity`；双击入口传入实体 handle。
- 用户输入参数：显示模式、最大三角边长、是否过滤退化三角形。
- 已有设计实体：已创建或导入的地形数模实体。
- 外部数据：无。

## 输出结果

- CAD 图形实体：更新后的 `DnTerrainTinEntity`。
- 领域实体：更新后的 `TinBuildOptions`、`TinTriangle` 和显示配置。
- 表格或报告：无。
- 更新通知或重建请求：第一版只更新当前数模实体。

## 操作流程

1. 用户运行 `DN_TERRAIN_TIN_EDIT` 或点击 Ribbon `编辑数模`。
2. 用户选择已有 `DnTerrainTinEntity`。
3. 用户也可双击数模实体，由托管 Ribbon 插件发送 `DN_TERRAIN_TIN_EDIT_HANDLE <handle>`。
4. C++ ObjectARX 侧打开数模编辑窗口。
5. 用户调整显示模式、最大三角边长和退化三角形过滤。
6. 系统使用实体内保存的点重新构网，并刷新实体显示。

## 关键业务规则

- 编辑流程不得丢失实体内保存的地形点和来源追踪信息。
- `ColoredTriangles` 在当前版本按三角网边线平均高程做 TrueColor 连续渐变映射，不填充三角面。
- `BoundaryOnly` 只绘制边界边。
- 双击入口只负责 handle 转发，实体读取和编辑逻辑仍由 C++ ObjectARX 侧完成。
- 当前编辑窗口仍有历史 C++ Win32 实现；后续新增或重做窗口必须迁移到 WPF。

## 可复用性说明

- 可复用内容：实体内点集重构网、显示模式切换、handle 编辑入口、双击转发协议。
- 临时原型内容：历史 C++ Win32 编辑窗口。
- 正式复用前需要改造的内容：WPF 参数窗口、源对象 reactor、自动重建和大数据性能优化。

## 与其他模块的依赖关系

- 上游模块：地形构网、RMesh 导入。
- 下游模块：纵断面取高、横断面、道路模型、土方计算。
- 实体联动行为：第一版暂不自动通知依赖对象重建。

## 后续对接正式 EICAD 功能的注意事项

- 编辑 UI 可替换，但 TIN 重建和显示规则必须继续由 C++ 核心承载。
- 后续应接入统一关系管理，数模变化后通知纵横断面等依赖对象。

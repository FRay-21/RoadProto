# 边坡模板与横断面戴帽边坡模型设计

## 目标

在横断面设计模块中新增“边坡模板”功能，并在“横断面戴帽”中使用边坡模板从路基模板最外侧开始放坡，结合 TIN 地形数模生成三维边坡线框。第一版继续沿用 V0.1 的线框模型原则，不生成边坡面、厚度、材质或算量。

## 范围

本次实现包含两个可连续验证的闭环：

1. 创建和编辑独立 `DnSlopeTemplateEntity` 边坡模板实体。
2. 在 `DnRoadModelEntity` 中保存并绘制边坡模型线，边坡结果与现有道路模型实体合并保存，但内部与路基部件线分组。

## 架构

沿用 RoadProto 分层：

- `domain/cross_section` 保存边坡模板数据、默认值、参数三选二规则、模板组优先级和放坡几何规则，不依赖 ObjectARX。
- `application/cross_section` 提供边坡模板默认创建服务和道路模型构建入口。
- `cad_adapter/objectarx/cross_section` 负责 CAD 点取、实体读写、TIN 实体读取、WPF 桥接、DWG 持久化和显示。
- `ui/wpf/RoadProto.Terrain.UI` 负责边坡模板窗口、横断面戴帽边坡 tab、模板组管理窗口和请求/响应文件。

## 边坡模板

新增用户命令：

- `RD_SECTION_SLOPE_TEMPLATE_CREATE`
- 显示名称：`创建边坡模板`
- Ribbon：`RoadProto / 横断面设计 / 创建边坡模板`
- 自定义实体：`DnSlopeTemplateEntity`
- DXF 名称：`DNSLOPETEMPLATEENTITY`

内部桥接命令：

- `RD_SECTION_SLOPE_TEMPLATE_EDIT_HANDLE`
- `RD_SECTION_SLOPE_TEMPLATE_APPLY_DIALOG_FILE`

模板属性：

- 名称，默认 `边坡模板1`
- 显示比例，默认 `1:10`
- 模板类型：`填方` 或 `挖方`
- 控制条件：
  - `交地则结束放坡`
  - `重复最后一组参数直至交地`

部件类型：

- `填方边坡`
- `挖方边坡`
- `护坡道`

每个部件支持三种编辑方式：

- 固定坡率和坡高，自动计算宽度。
- 固定坡率和宽度，自动计算坡高。
- 固定坡高和宽度，自动计算坡率。

每个部件都有 `搜索地面线范围坡高增值`。为空或 `0` 不生效；大于 `0` 时，按当前部件方向继续试探指定竖向高差绝对值，若试探范围内交地，则当前部件直接延伸到交点。

默认模板：

- 填方：边坡、护坡道、边坡、护坡道、边坡；边坡坡率 `1:1.5`、坡高 `4`、坡高增值 `2`；护坡道宽度 `1`、横坡 `3%`。
- 挖方：同样五段，边坡方向反向；边坡坡率 `1:-1.5`，坡高 `4`，坡高增值 `2`；护坡道方向随挖方反向。

## 横断面戴帽

扩展 `横断面戴帽` 窗口：

- 全局参数：
  - 采样间隔。
  - 左侧搜索宽度，默认 `50`。
  - 右侧搜索宽度，默认 `50`。
- 一级 tab 新增 `边坡模板`。
- `边坡模板` 内二级 tab：
  - `左侧边坡模板`
  - `右侧边坡模板`

每侧配置多个模板组：

- 模板组行有起点桩号、终点桩号、自动序号、组内模板摘要。
- 行越靠上优先级越高。
- 组内可包含多个边坡模板，按从上到下尝试。
- 先戴帽成功的模板被采用。
- 左右侧独立搜索、独立成功或失败。

## 戴帽成功规则

边坡从当前桩号、当前侧的路基模板最外侧边界点开始。搜索宽度从道路中线量起，左侧不超过 `leftSlopeSearchWidth`，右侧不超过 `rightSlopeSearchWidth`。

成功判定：

- 两个控制条件都关闭：部件参数合法，按模板生成完成且不超过该侧搜索宽度，即戴帽成功，不要求交地。
- `交地则结束放坡` 开启：理论部件段或坡高增值试探段一旦交地，截到交点并成功结束；如果模板走完未交地且未开启重复，仍然成功。
- `重复最后一组参数直至交地` 开启：模板走完仍未交地时，必须重复“最后一个护坡道 + 紧随其后的最后一个边坡”直到交地；超过搜索宽度仍未交地则失败。
- 两个控制条件同时开启：先执行“交地则结束”，模板走完仍未交地时再重复最后一组直到交地。
- 找不到“护坡道 + 边坡”最后一组时，开启重复的模板无效，当前模板失败并尝试组内下一个模板。
- 如果当前桩号没有路基模板最外侧边界，该侧直接失败并跳过。

## TIN 横断面剖切

为兼顾准确和性能，第一版采用 TIN 横断面精确剖切并带轻量候选过滤：

1. 每次生成道路模型时，把 TIN 点、三角形和三角形 XY 包围盒整理为轻量缓存。
2. 每个采样桩号按道路中线法线剖切一次地面线，左右侧共用剖切结果。
3. 剖切范围为 `[-leftSlopeSearchWidth, rightSlopeSearchWidth]`。
4. 先用三角形 XY 包围盒过滤候选，再计算横断面线与三角形边的交点。
5. 交点高程按三角面内插。
6. 地面折线按偏距排序并去重。
7. 边坡部件线与地面折线做二维线段求交。

若实际测试在大 TIN 上明显卡顿，后续再升级为空间索引；本次保留缓存接口，避免把首版复杂度压得过高。

## 验收

自动测试：

- 核心测试覆盖边坡模板默认值、三选二计算、模板组优先级、成功/失败规则、TIN 剖切交地和道路模型边坡线输出。
- 托管桥接测试覆盖边坡模板请求/响应、道路模型新增边坡字段和模板组文件读写。
- 源码契约测试覆盖 ObjectARX 实体、命令注册、Ribbon、双击编辑和项目文件包含。

手工验证：

- 创建边坡模板后点取插入点，WPF 窗口显示默认模板和预览。
- 修改参数后预览刷新，确认后图中生成 `DnSlopeTemplateEntity`。
- 双击边坡模板可重新打开并保留参数。
- 横断面戴帽中配置路基模板、左右边坡模板组、搜索宽度，生成后 `DnRoadModelEntity` 显示路基和边坡三维线框。

# 路基模板设计说明

日期：2026-05-12

## 目标

在 Ribbon 中新增“横断面设计”模块，并提供“创建路基模板”功能。用户运行命令后只点取一个插入点，不选择、不绑定现有道路中线；系统创建一个独立路基模板自定义实体，实体自身绘制道路中线标记，并预留后续与道路中线、路面结构层关联的字段。

## 功能范围

- 新增 Ribbon 模块：`横断面设计`。
- 新增命令：`RD_SECTION_SUBGRADE_TEMPLATE_CREATE`，按钮文字为“创建路基模板”。
- 新增自定义实体：`DnSubgradeTemplateEntity`，DXF 名称为 `DNSUBGRADETEMPLATEENTITY`。
- 运行命令后：
  - 用户在图中点取插入点。
  - 弹出 WPF 参数对话框。
  - 用户确认后，将模板实体写入 DWG。
- 双击 `DnSubgradeTemplateEntity` 后，重新打开同一个 WPF 对话框编辑参数。

## 参数模型

模板全局参数：

- 路基模板名称。
- 显示比例：`1:1`、`1:10`、`1:20`、`1:50`、`1:100`。
- 道路等级：高速公路、一级道路、二级道路、三级道路、四级道路、城市快速路、城市主干道、城市次干道、城市支路。

部件参数：

- 侧别：左侧或右侧。左侧部件只能留在中线左侧，右侧部件只能留在中线右侧。
- 类型：中分带、行车道、硬路肩、土路肩、侧分带、人行道、慢车道、路缘带。
- 宽度。
- 变宽数据表：桩号、加宽值。实际宽度为基础宽度加加宽值；当前实体未绑定道路中线，预览和 CAD 图形先按基础宽度显示，表格数据持久化保留。
- 颜色。
- 高度：部件起点相对内侧部件的高度差，可正可负。
- 坡度：支持固定值和变化值。变化值表为桩号、坡度；选择变化值后固定坡度输入置灰，预览和 CAD 图形只从坡度变化表取值，不再回退固定坡度。
- 关联路面结构层：本版先预留界面和数据字段；启用后才允许填写厚度，后续再连接路面结构层自定义实体。

## 初始模板

高速公路：

- 左侧中分带 `1.5`
- 左侧路缘带 `0.75`
- 左侧行车道 `3.75 * 2`
- 左侧硬路肩 `3`
- 左侧土路肩 `0.75`
- 右侧对称

城市快速路：

- 左侧中分带 `1`
- 左侧行车道 `3.75 * 2`
- 左侧侧分带 `1`
- 左侧慢车道 `3`
- 左侧人行道 `4`
- 右侧对称

二级、三级、四级道路：

- 左侧行车道 `3.75`
- 左侧行车道 `3.75`
- 左侧硬路肩 `3`
- 左侧土路肩 `0.75`
- 右侧对称

一级道路、城市主干道、城市次干道和城市支路本版先生成空模板，后续按业务规则扩展默认部件。

## 颜色约定

颜色初始值写死，并保持每次生成一致：

- 中分带、侧分带：绿色，低饱和深色 RGB。
- 行车道、硬路肩、慢车道、路缘带：蓝色，低饱和深色 RGB。
- 土路肩、人行道：灰色。

## 架构边界

- `domain/cross_section`：保存路基模板枚举、部件模型、默认模板和轻量校验，不依赖 ObjectARX 或 WPF。
- `application/cross_section`：生成创建命令所需的默认模板数据。
- `modules/cross_section`：通过 `CommandRegistry` 注册命令，通过 `ModuleRegistry` 注册模块。
- `cad_adapter/objectarx/cross_section`：负责点取插入点、DWG 实体、DWG 持久化、绘制、WPF Bridge 文件交互和双击编辑入口。
- `ui/wpf`：只负责参数编辑、二维预览、二级表格和回写 Bridge 文件，不直接操作 `AcDbEntity`、`AcDbObjectId` 或 `ads_name`。

## 后续预留

- 与道路中线实体建立关联后，可按桩号解析变宽表和坡度变化数据表，并统一向中线外侧加宽。
- 与路面结构层实体建立关联后，可从结构层实体读取厚度、层名、材料等参数。
- 与统一关系管理机制连接后，可在道路中线或路面结构层变化时标记路基模板实体需要重建。

# 纵断面拉坡图复用说明

## 能力范围

纵断面拉坡图能力用于从道路中线、地形数模或 DMX 纵地面线文件生成可持久化的纵断面图形实体。当前沉淀的可复用能力分为四类：

- DMX 纵地面线文件读取。
- 拉坡图数据模型和布局映射。
- 拉坡图创建服务。
- ObjectARX 自定义实体和 WPF 属性 Bridge。

## 可复用能力

### DMX 文件读取

- 源码：`src/domain/profile/ProfileDmxFile.*`
- 输入：`.dmx` 文件路径。
- 输出：`ProfileGroundSample` 列表和错误信息。
- 规则：跳过空行和 `//` 注释；读取 `ZH H` 两列；兼容 `37123.456_2` 断链写法；保留重复桩号；拒绝非有限数值和无效行。
- 边界：断链号当前不参与分段排序和断链换算，只保留桩号数值部分用于布局。

### 拉坡图模型和布局

- 源码：`src/domain/profile/ProfileGradeGraphModel.h`、`src/domain/profile/ProfileGradeGraphLayout.*`
- 数据：来源类型、道路名称、道路 handle、数模 handle、DMX 路径、地面线样本、显示属性。
- 布局：X 方向按桩号 1:1；Y 方向按纵向比例映射；根据样本高程和网格间距计算基准高程、顶部高程、图宽和图高。
- 约束：样本不少于两个；桩号范围必须大于 0；网格间距必须为正；地面线宽度当前不超过 `2.11mm`；纵向比例当前只接受 `1`、`10`、`100`。

### 创建服务

- 源码：`src/application/profile/ProfileGradeGraphCreateService.*`
- 职责：把道路名称、来源信息、地面线样本和默认属性组装为 `ProfileGradeGraphData`。
- 默认值：图名为“道路名称 + 拉坡图”；道路名称为空时为“道路拉坡图”；地面线颜色 ACI 4；线宽 `1`；精度 `10`；纵向比例 `10`；网格间距 `10`。
- 边界：服务不读取 CAD 实体，不弹文件框，不调用数模高程查询。

### 自定义实体

- 源码：`src/cad_adapter/objectarx/profile/DnProfileGradeGraphEntity.*`
- 职责：保存拉坡图数据、插入点和局部坐标轴；绘制标题、表头、网格线、高程/桩号标注和地面线折线；支持 DWG 读写、几何范围和变换。
- 持久化：保存图名、来源类型、道路/数模 handle、DMX 路径、样本列表和显示属性。
- 边界：实体只负责表达已有数据；路线移动后的自动重建由后续关系管理机制触发，不在实体内部监听其他对象。

### WPF 属性 Bridge

- 源码：`src/cad_adapter/objectarx/profile/ProfileGradeGraphDialogBridge.*`、`src/ui/wpf/RoadProto.Terrain.UI/ProfileGradeGraphWindow.xaml`
- 职责：C++ 写请求文件、托管命令打开 WPF、WPF 写响应文件、C++ 读取并回写实体。
- 可编辑属性：名称、地面线颜色、线宽、精度、纵向比例、网格间距。
- 更新地面线：DMX 来源按钮启用并重新读取实体保存的 DMX 路径；数模来源按钮置灰，但修改精度并确定时会按保存的道路中线和数模 handle 重新采样。
- 边界：当前 Bridge 是原型文件桥接，后续可替换为更正式的进程内或消息式 Bridge，但 WPF 仍不直接操作 ObjectARX 类型。

## 适用场景

- 纵断面地面线显示。
- 后续竖曲线设计前的基础拉坡图底图。
- 从外部 DMX 数据快速生成纵断面图。
- 与道路中线和地形数模联动的后续设计对象基础。

## 后续扩展

- 通过实体关系管理记录道路中线、数模和拉坡图之间的依赖。
- 路线移动或数模更新后自动标记拉坡图需要重建。
- 增加竖曲线设计线、坡度坡长表、变坡点、填挖高度和更多纵断面标注。
- 完善断链桩号分段表达，而不是只读取数值部分。

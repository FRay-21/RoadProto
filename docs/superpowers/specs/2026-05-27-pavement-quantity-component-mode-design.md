# 路面工程量统计表部件模式设计

## 背景

`RD_DRAWING_PAVEMENT_QUANTITY_TABLE` 已能按构造物范围切分普通段、桥梁段和隧道段，并按结构层类型输出面积和体积。用户希望在输出面积、体积列时增加部件名称，例如 `行车道`，用于区分同一结构层类型在不同行车部件中的工程量；同时保留上一版只按结构层类型汇总的输出方式。

## 设计

- 统计模式增加两种：
  - `按部件和结构层`：动态列使用 `部件名称-结构层名称面积`、`部件名称-结构层名称体积`。
  - `按结构层类型`：动态列保持上一版 `结构层名称面积`、`结构层名称体积`。
- 默认模式为 `按部件和结构层`。
- 部件名称来自道路模型生成结构层节点时对应的路基模板部件类型中文名，例如 `行车道`、`硬路肩`、`土路肩`。
- 新生成的道路模型和横断面落图会保存结构层部件名称；旧实体缺少部件名称时使用 `未分部件` 作为兼容名称。
- 输出路径选择优先使用带统计方式选项的 Windows 保存文件对话框；若该对话框不可用，命令退回 AutoCAD 文件选择，并使用默认统计模式。

## 数据流

1. 道路模型构建结构层断面节点时，把路基部件类型中文名写入 `RoadModelSectionNode::componentName`。
2. 查看横断面预览和绘制横断面时，把部件名称写入结构层面域 `RoadModelSectionDrawingFace::componentName`。
3. 工程量命令读取道路模型断面节点，优先用道路模型中的部件名称；无法读取道路模型时，用横断面图面域保存的部件名称。
4. `PavementQuantityTableCalculator` 按用户选择的统计模式聚合样本。

## 测试

- 核心测试覆盖同一结构层在 `行车道` 和 `硬路肩` 中按部件拆列。
- 核心测试覆盖 `按结构层类型` 仍合并为上一版结构层列。
- 源码契约测试覆盖保存对话框包含统计方式选项。
- 源码契约测试覆盖道路模型实体和横断面落图实体持久化部件名称。

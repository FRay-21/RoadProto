# 路面结构层模板_创建

## 基本信息

- 功能名称：路面结构层模板创建
- 所属模块：横断面设计
- 命令名称：`RD_SECTION_PAVEMENT_LAYER_TEMPLATE_CREATE`
- 对应代码入口：`src/cad_adapter/objectarx/cross_section/ObjectArxPavementLayerTemplateCommand.cpp`
- 业务文档维护人：RoadProto 项目
- 原型版本：`v0.1.24`
- 是否可复用：部分可复用

## 功能背景

横断面道路模型需要把路基模板部件与可复用的路面结构层参数分离。路面结构层模板作为独立 DWG 实体保存层类型、厚度、内外侧加宽和横坡信息，供多个路基模板部件引用，并可通过 `.rpavement.xml` 在图纸或项目间流转。

## 业务目标

- 在 CAD 图中创建独立 `DnPavementLayerTemplateEntity`。
- 通过 WPF 参数窗口维护模板名称、显示比例、预览宽度和结构层列表。
- 支持结构层类型下拉值：上面层、中面层、下面层、沥青封层、基层、底基层、垫层、搭板层。
- 运行创建命令后先打开路面结构层创建向导，选择路面类型和适应路段类型，并可在同一层行中快速调整文档预设中的各层名称、内外侧厚度、内外侧加宽和内外侧坡度。
- 向导确认后继续打开原有 WPF 参数窗口；双击或按 handle 编辑既有实体时不显示创建向导。
- 支持每一层独立编辑 RGB 真彩色；边线使用该层颜色，填充使用同色半透明效果。
- 支持每一层选择 CAD 常用填充类型，并输入填充角度和填充比例；通过通用“显示方式”在按颜色、按填充、按填充+颜色显示之间切换；该显示方式仅影响 WPF 预览和路面结构层模板自定义实体。
- 所有路面结构层类型的预览宽度初始默认值统一为 `3`。通用参数区默认只显示预览宽度、结构层数量、显示方式和当前编辑层；勾选“显示全部通用参数”后才显示结构代号、路基干湿类型、路面类型、路基土组、设计弯沉和累计轴次。新增通用参数仅作为 `PavementLayerTemplateProperties` 数据保留，暂不参与预览标注、DWG 模板实体文字或道路模型结构层显示。
- WPF 参数区按“当前编辑层”显示单层参数，预览图点击可选中结构层，也可通过当前层输入框和上/下按钮切换；新增部件时询问在当前部件上方或下方插入，删除部件时询问是否删除当前选中部件。
- 支持“内外厚度是否一致”复选框；勾选时使用单一厚度，未勾选时分别填写内侧厚度和外侧厚度。
- 支持“内外加宽是否一致”和“内外坡度是否一致”复选框；默认勾选，一致时使用单一值，未勾选时分别填写内侧和外侧。
- 明确内外侧语义：内侧是相对路基部件更靠近道路中线的一侧，外侧是更远离道路中线的一侧；内侧加宽为正时向道路中线方向扩宽，为负时向远离道路中线方向缩短当前层。
- 支持 WPF 窗口导入/导出 `.rpavement.xml`。

## 适用场景

适用于需要在横断面路基模板中复用同一套路面结构层的原型图纸。模板实体本身不绑定道路中线，可被任意路基模板部件引用。

## 输入条件

- CAD 选择对象：无，用户只点取模板插入点。
- 用户输入参数：创建向导中的路面类型、适应路段类型、层名、内外侧厚度、内外侧加宽和内外侧坡度；原有参数窗口中的模板名称、显示比例、预览宽度、显示方式、结构层类型、层名、RGB 颜色、填充类型、填充角度、填充比例、厚度、内外侧加宽、内外侧坡度，以及勾选“显示全部通用参数”后可维护的结构代号、路基干湿类型、路面类型、路基土组、设计弯沉和累计轴次。
- 已有设计实体：无。
- 外部数据：可选 `.rpavement.xml`。

## 输出结果

- CAD 图形实体：`DnPavementLayerTemplateEntity`
- 领域实体：`PavementLayerTemplateData`
- 表格或报告：无
- 更新通知或重建请求：当前版本不自动触发引用方重建。

## 操作流程

1. 运行 `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_CREATE` 或点击 Ribbon `横断面设计 / 创建路面结构层模板`。
2. C++ 应用服务生成默认 `PavementLayerTemplateData`，并以 `showCreateWizard=true` 写入 WPF 请求文件。
3. WPF 先打开路面结构层创建向导；用户选择路面类型和适应路段类型后，系统按文档预设生成初始结构层参数。
4. 用户在向导中可快速调整各层材料名称、内外侧厚度、内外侧加宽和内外侧坡度；取消向导时不创建 DWG 实体。
5. 用户确认向导后，WPF 继续打开原有路面结构层模板参数窗口；用户可细化层列表，也可导入 `.rpavement.xml` 覆盖当前窗口数据。
6. 用户确认原有参数窗口后，WPF 写出响应文件并发送 `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_APPLY_DIALOG_FILE`。
7. C++ 回写命令提示用户在 CAD 图中点取模板插入位置，随后创建 `DnPavementLayerTemplateEntity`，写入插入点和模板数据，并追加到模型空间。

## 关键业务规则

- 路面结构层类型只能使用：上面层、中面层、下面层、沥青封层、基层、底基层、垫层、搭板层；桥接和 XML 中使用稳定枚举编码保存。
- 创建向导的路面类型包括沥青路面、混凝土路面；沥青路面适应路段类型包括主线行车道、主线硬路肩、互通匝道、桥头过渡段、桥面铺装、互通被交路、隧道；混凝土路面适应路段类型包括收费站广场、通道连接线。
- 创建向导中任意路面类型和适应路段类型生成的模板，初始预览宽度均为 `3`。
- 沥青路面主线行车道预设中，基层和底基层的内侧加宽为 `0.1`、内侧坡度为 `1`，外侧加宽和外侧坡度均为 `0`；沥青路面主线硬路肩预设中，基层和底基层的外侧加宽为 `0.1`、外侧坡度为 `1`，内侧加宽和内侧坡度均为 `0`。未特别指定的已有加宽和坡度预设按内外侧一致保存。
- 文档预设中空白或 0 的层参数按“没有”处理，不生成有效结构层参数；例如互通匝道和桥头过渡段中的空白下面层不进入初始层列表。
- 文档预设中的填充类型和填充比例同属默认参数。若 `参数示意.docx` 的“填充类型”单元格使用图片表达，应按图片上标注的 AutoCAD 填充名读入，不得因为文本抽取为空而回退为 `SOLID`。当前材料名称到默认填充的映射如下：
- 向导确认后克隆到路面结构层参数窗口时，合法的文档填充名必须继续保留；只有不在候选列表中的非法填充名才允许归一化为 `SOLID`，不得把 `NET`、`AR-HBONE`、`GRAVEL`、`SACNCR`、`TRIANG`、`AR-SAND`、`HEX` 等文档默认值二次回退。
- WPF 预览图的层名和厚度使用白色引线式标注：一根竖向引线从结构层顶边下引，起点不得超出路面顶部；右侧按结构层顺序一层一行显示名称和厚度，每行配一条不过长的白色下划线。文字、引线、下划线、加宽箭头和标注偏移均使用固定模型尺寸，随预览缩放按比例显示，不保持屏幕像素大小不变；加宽尺寸线和箭头保持短小，加宽文字缩小并标在尺寸箭头中部上方，坡度文字缩小并标在坡边中部侧边。

- 每一层必须有正厚度；等厚模式保存 `thickness`，非等厚模式保存 `innerThickness` 和 `outerThickness`。
- 内侧和外侧按路基部件解释，不按 CAD 图纸左/右绝对方向解释。
- 内侧加宽为正时扩向道路中线，外侧加宽为正时扩向远离道路中线的一侧；加宽为负时表示当前层顶边相对上一层底边所在直线向内缩短。加宽只表达当前层自身顶边长度变化，不代表修改上一层坡率，也不得生成台阶折线。若上一层底边宽度为 7.5m，下一层内外侧各加宽 1m，则下一层顶边总宽度为 9.5m；若两侧加宽为 -0.5m，则当前层顶边总宽度比上一层底边缩短 1m。
- 每一层结构层按顶边、底边、内侧边和外侧边四条边表达。顶边就是图形上方边，底边就是图形下方边；默认坡度为 0 时，顶边到底边的侧边竖直；输入坡度后，内侧边和外侧边应从顶边向下生成到底边。
- 除第一层外，当前层顶边必须以上一层底边所在直线为基准；本层加宽沿该直线向两端平行/共线延长或收回。当前层始终是四边形/梯形，上一层底边与当前层顶边的重叠线段保持共线，不允许用“接触边 + 外轮廓”的六点台阶形表达。
- 坡度只用于计算当前层底边相对当前层顶边的水平位移：顶边先按加宽确定，底边再根据对应侧厚度和 `1:n` 坡度计算。坡度输入支持 `1:n` 或数字 `n`，其中 `n` 表示每 1 个竖向厚度对应的水平位移量；水平位移量 = `厚度 * n`。正坡度表示从顶边向下到底边时侧边向外放，负坡度表示向内收。`1:1` 时侧边与底边夹角约为 45°，`1:2` 时约为 30°，`1:0.5` 时约为 60°。允许负坡度，`1:-0.5` 时侧边向内收，侧边与底边夹角约为 120°。坡度为 0 时顶边到底边的侧边竖直。
- 修改厚度、加宽或坡度后，WPF 预览、`DnPavementLayerTemplateEntity` 图面显示、道路模型结构层三维填充面/边线和查看横断面结构层预览都必须使用同一套四边形/梯形轮廓表达；不得让模型或 DWG 实体继续使用六点台阶轮廓。
- 每一层保存独立 RGB 真彩色。默认新建模板仍按层号给出蓝、绿、黄、橙、紫、灰初始色，但用户修改后，WPF 预览、`DnPavementLayerTemplateEntity` 图面显示、道路模型结构层填充面/边线和查看横断面结构层预览必须使用该层保存的 RGB，不再按层号临时取色，也不继承路基模板部件颜色。
- WPF 预览、`DnPavementLayerTemplateEntity` 和 `DnRoadModelEntity` 的结构层显示应先绘制所有结构层填充，再绘制边线和标注/线框；DWG 模板实体和道路模型结构层面都必须使用与 WPF 相同的四点轮廓顺序绘制 `polygon` 填充，但填充色使用按 WPF 预览背景和透明度预混合后的弱化 RGB，避免 AutoCAD 真实透明、视觉样式或代理图形缓存把颜色显示得过亮或形状分叉。边线必须使用层保存 RGB，文字标注使用支持中文的文字样式。WPF 预览保留层名/厚度、加宽和坡度标注，DWG 模板实体不显示尺寸标注，只在模板上方按文字总长度居中显示模板名称。当相邻层边界共线重叠时，必须沿同一几何线表达，避免左右厚度不一致时斜向边界重复或台阶化而看起来像层间相交。
- WPF 预览初始应居中显示结构层模板；鼠标滚轮缩放以鼠标当前位置为基点。
- WPF 只负责参数编辑、预览和 XML 文件读写；模板数据归一化由 C++ `PavementLayerTemplateRules` 执行。
- `structureCode`、`subgradeMoistureTypes`、`pavementType`、`subgradeSoilGroups`、`designDeflection` 和 `cumulativeAxleLoads` 当前只做数据持久化和 XML/Bridge 流转，不改变结构层几何、颜色、填充或标注。
- `.rpavement.xml` 用于模板参数流转，不代表 DWG 实体持久化格式。

## 可复用性说明

- 可复用内容：`PavementLayerTemplateModel`、`PavementLayerTemplateRules`、`PavementLayerTemplateCreateService`、`DnPavementLayerTemplateEntity`
- 临时原型内容：WPF 请求/响应文件 Bridge、当前命令中的插入点点取和弹窗调起流程
- 正式复用前需要改造的内容：模板库、结构层材料库、统一实体关系和引用方自动重建

## 与其他模块的依赖关系

- 上游模块：无
- 下游模块：横断面路基模板、横断面道路模型、查看横断面
- 实体联动行为：当前版本由路基模板部件保存结构层模板 handle，道路模型生成时读取；模板更新后需要重新生成道路模型。

## 后续对接正式 EICAD 功能的注意事项

- 后续应增加材料、压实度、造价、分层编号等正式结构层属性。
- 后续应接入统一关系管理机制，提示引用该模板的路基模板或道路模型需要重建。

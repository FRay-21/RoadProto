# 道路模型_编辑

## 基本信息

- 功能名称：道路模型编辑
- 所属模块：横断面设计
- 命令名称：`RD_SECTION_ROAD_MODEL_EDIT`、`RD_SECTION_ROAD_MODEL_EDIT_HANDLE`
- 对应代码入口：`src/cad_adapter/objectarx/cross_section/ObjectArxRoadModelCommand.cpp`
- 业务文档维护人：RoadProto 项目
- 原型版本：`v0.1.13`
- 是否可复用：部分可复用

## 功能背景

道路模型创建后，需要允许用户重新调整模板生效范围、模板优先级和采样间距，并基于原道路中线与竖曲线重新生成三维部件线。

## 业务目标

- 选择已有 `DnRoadModelEntity` 并重新打开横断面戴帽窗口。
- 双击道路模型实体时通过 handle 命令进入同一编辑窗口。
- 保留实体中已保存的中线 handle、竖曲线 handle、采样间距和模板范围表。
- 编辑时仍可在每一行点选图中的路基模板实体，回填模板 handle 和名称。
- 确认后重新构建并刷新同一个道路模型实体。

## 适用场景

适用于已生成道路模型后需要二次调整模板范围或模板优先级的原型图纸。

## 输入条件

- CAD 选择对象：`DnRoadModelEntity`
- 用户输入参数：采样间距、模板范围表、模板点选结果、模板优先级顺序
- 已有设计实体：实体保存的中线、竖曲线和模板 handle 指向的对象仍存在
- 外部数据：无

## 输出结果

- CAD 图形实体：更新后的 `DnRoadModelEntity`
- 领域实体：重新构建后的 `RoadModelData`
- 表格或报告：无
- 更新通知或重建请求：当前版本不自动接入统一关系重建

## 操作流程

1. 运行 `RD_SECTION_ROAD_MODEL_EDIT`，或双击道路模型实体触发 `RD_SECTION_ROAD_MODEL_EDIT_HANDLE`。
2. 系统读取道路模型实体内保存的配置。
3. 打开同一个 WPF `横断面戴帽` 窗口，并显示上次保存的路基模板范围表。
4. 用户修改采样间距、桩号范围、模板 handle 或表格行顺序，也可在某一行点选图中路基模板实体。
5. 点击 `生成模型`。
6. C++ ObjectARX 重新读取中线、竖曲线和模板实体，调用 `RoadModelBuildService`。
7. 系统用新的 `RoadModelData` 刷新原 `DnRoadModelEntity`。

## 关键业务规则

- 编辑命令只更新选中的道路模型实体，不创建新实体。
- 如果保存的中线、竖曲线或模板 handle 已失效，回写失败并提示错误。
- 行内点选模板只更新对应行的模板引用，不改变道路模型其他保存配置。
- 竖曲线必须通过其拉坡图关联到当前道路中线，否则不允许构建。
- 旧模型中的三维部件线由重新构建结果整体替换，避免保留过期线段。

## 可复用性说明

- 可复用内容：`RoadModelData` 配置持久化、WPF 请求/响应 DTO、`RoadModelBuildService`
- 临时原型内容：双击 handle 通过托管 Ribbon 事件转发命令
- 正式复用前需要改造的内容：实体关系追踪、批量重建、模板选择 UI 和变更提示

## 与其他模块的依赖关系

- 上游模块：平面设计、纵断面设计、横断面路基模板
- 下游模块：三维模型、出图、算量
- 实体联动行为：当前版本只保存 handle，不自动监听上游修改

## 后续对接正式 EICAD 功能的注意事项

- 需要给道路模型增加更明确的属性窗口、模板列表选择器和错误定位提示。
- 后续应支持上游对象变更后的自动标脏和重建请求。

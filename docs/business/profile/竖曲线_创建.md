# 竖曲线创建

> 文档粒度要求：本文只描述竖曲线创建入口，不描述通用属性编辑、夹点移动或 PVI 删除。

## 基本信息

- 功能名称：竖曲线创建
- 所属模块：纵断面设计
- 命令名称：`RD_PROFILE_VERTICAL_CURVE_CREATE`
- 对应代码入口：`ObjectArxProfileVerticalCurveCommand.*`、`ProfileVerticalCurveCreateService.*`、`DnProfileVerticalCurveEntity.*`
- 业务文档维护人：RoadProto
- 原型版本：V0.1.9 follow-up
- 是否可复用：是

## 功能背景

纵断面拉坡图负责表达地面线和图面坐标系，竖曲线用于表达道路纵断面上的设计高程线。竖曲线采用独立 ObjectARX 自定义实体，保存所属拉坡图 handle、起终点控制点、PVI 列表、半径和显示属性，后续可被横断面、三维模型、土方和排水设计引用。

## 业务目标

用户运行创建命令后选择一个纵断面拉坡图，系统默认生成一条连接地面线起终点的竖曲线实体。初始实体没有 PVI，表现为设计直线，后续可通过夹点、PVI 命令或 WPF 编辑窗口继续调整。

## 输入条件

- CAD 选择对象：`DnProfileGradeGraphEntity`。
- 用户输入参数：选择集或点选拉坡图实体。
- 已有设计实体：纵断面拉坡图必须已有不少于两个地面线样本。
- 外部数据：无。

## 输出结果

- CAD 图形实体：`DnProfileVerticalCurveEntity`。
- 领域数据：`ProfileVerticalCurveData`，包含起点、终点、PVI 列表和显示属性。
- 默认图形：连接拉坡图首末地面线样本的设计直线，直坡段采用青色显示。
- 更新通知：当前直接刷新实体显示；自动联动重建后续接入统一关系管理机制。

## 操作流程

1. 用户点击 Ribbon `RoadProto / 纵断面设计 / 创建竖曲线`，或运行 `RD_PROFILE_VERTICAL_CURVE_CREATE`。
2. 命令提示选择纵断面拉坡图实体。
3. C++ CAD adapter 读取拉坡图 handle 和地面线样本。
4. `ProfileVerticalCurveCreateService` 使用地面线首末样本生成默认起终点控制点。
5. 命令插入 `DnProfileVerticalCurveEntity`，实体通过关联拉坡图的图面坐标系绘制。

## 关键业务规则

- 竖曲线实体独立于拉坡图实体，拉坡图只作为底图和坐标映射来源。
- 起点桩号、高程默认取首个地面线样本；终点桩号、高程默认取最后一个地面线样本。
- 默认 PVI 列表为空，表示起终点之间的直坡设计线。
- 图形表达按设计线段类型分色：直坡设计线为青色，竖曲线曲线段为黄色。
- 当存在 PVI 时，系统绘制 BVC/PVC - PVI - EVC/PVT 的理论切线，默认采用白色细线。
- 竖曲线桩号范围不强制裁剪到拉坡图地面线范围；后续拖动起终点可比拉坡图长或短。
- WPF 不直接操作 ObjectARX 类型，创建、持久化和图面刷新由 C++ 侧完成。

## 与其他模块的依赖关系

- 上游模块：纵断面拉坡图、平面设计、地形数模。
- 下游模块：竖曲线编辑、夹点编辑、横断面设计高程、三维道路模型、土方和排水设计。
- 联动边界：当前保存拉坡图 handle；路线或地形变化后的自动重建需后续通过实体关系管理表达。

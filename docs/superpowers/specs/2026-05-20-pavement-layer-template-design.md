# 横断面路面结构层模板设计

## 背景

横断面路面结构层模板用于定义一个横断面部件的路面结构层组成。用户先在 DWG 中创建独立的结构层模板实体，再在路基模板的任意部件上选择该模板。再次生成横断面道路模型时，系统应在原有路基、边坡三维线框基础上，把绑定部件下方的结构层三维线框一并生成出来。

本功能属于横断面设计模块，必须遵守 RoadProto 既有边界：领域模型和生成规则放在 C++ `domain/application` 层；ObjectARX 只负责 CAD 点取、实体持久化和图形显示；WPF 只负责参数展示、编辑、XML 文件交互和预览。

## 目标

- 新增独立命令创建“横断面路面结构层模板”。
- 新增独立 `DnPavementLayerTemplateEntity`，支持 DWG 保存重开、图面预览、插入点夹点移动和双击编辑。
- 新增 WPF 参数窗口，布局采用参考图同构形式：左侧大预览，右侧分组参数。
- 支持 XML 导入导出模板参数。
- 路基模板所有部件均可选择路面结构层模板。
- 横断面道路模型再次生成时，按路基模板部件绑定的结构层模板生成结构层三维线框。
- 查看横断面预览中显示结构层线。

## 命令与实体

新增用户命令：

- `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_CREATE`
- 显示名称：创建路面结构层模板
- Ribbon 位置：`RoadProto / 横断面设计 / 创建路面结构层模板`
- 业务文档：`docs/business/cross_section/路面结构层模板_创建.md`

新增内部桥接命令：

- `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_EDIT_HANDLE`
- `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_APPLY_DIALOG_FILE`

新增自定义实体：

- 类名：`DnPavementLayerTemplateEntity`
- DXF 名称：`DNPAVEMENTLAYERTEMPLATEENTITY`
- 保存内容：插入点、模板名称、显示比例、默认预览宽度、结构层列表
- 图面表达：只绘制单个模板预览图，不沿道路中线生成模型

## 数据模型

新增 `src/domain/cross_section/PavementLayerTemplateModel.*`。

模板数据：

- 模板名称
- 显示比例
- 默认预览宽度
- 结构层列表

结构层类型为固定枚举，下拉选择：

- 上面层
- 中面层
- 下面层
- 基层
- 底基层
- 垫层

每一层参数：

- 类型
- 名称
- 内外厚度是否一致，默认勾选
- 厚度，勾选“内外厚度是否一致”时显示
- 内侧厚度、外侧厚度，取消勾选时显示
- 内侧加宽
- 外侧加宽
- 内侧坡度
- 外侧坡度

内外定义：

- 内侧：同一个路基部件中更靠近道路中线的一侧。
- 外侧：同一个路基部件中更远离道路中线的一侧。
- 结构层模板参数始终按内/外填写；绑定到左侧或右侧路基部件时，由道路模型生成逻辑自动换算 CAD 坐标方向。

## 几何规则

结构层模板预览和道路模型生成使用同一套核心几何规则。

- 基础宽度来自绑定的路基模板部件当前桩号宽度。
- 在独立结构层模板窗口中，使用默认预览宽度显示，不标注宽度。
- 第 1 层上缘贴合路基部件顶线。
- 每层厚度向下生成。
- 每一层的内侧加宽、外侧加宽都是基于上一层边界的外扩量。
- 采用“加宽优先”规则：加宽决定层间边界位置；内侧坡度、外侧坡度用于侧边斜线表达和标注，不额外叠加宽度。
- 若勾选“内外厚度是否一致”，该层内外厚度相同。
- 若取消勾选，内侧厚度和外侧厚度分别控制该层内外端的下边界高程，层底线可形成横向倾斜。
- 坡度输入 `1` 表示工程写法 `1:1`。

## WPF 窗口

窗口布局采用用户确认的 A 方案：参考图同构布局。

顶部：

- 模板名称输入框
- 导入按钮
- 保存按钮
- 显示比例输入框或下拉输入

左侧：

- 大面积结构层模板预览图
- 支持鼠标滚轮缩放
- 支持鼠标中键按住拖动平移
- 根据参数实时刷新
- 显示每层厚度、内侧加宽、外侧加宽
- 某侧坡度为 `0` 或空时不标注该侧坡度
- 标注采用分区避让：内侧标注靠近道路中线一侧，外侧标注靠远离道路中线一侧，厚度标注放层中部或侧边；层数多时可缩小字体并拉开引线，优先避免重叠

右侧：

- 常规分组：路面结构层模板名称、层数
- 每层结构参数分组：类型、名称、厚度一致复选框、厚度或内外厚度、内侧加宽、外侧加宽、内侧坡度、外侧坡度
- 层数改变时，参数分组数量随之增减

底部：

- 确定：回写 DWG 实体并关闭窗口
- 取消：不回写
- 应用：回写 DWG 实体但保持窗口

顶部 XML 按钮语义：

- 保存：导出当前模板参数到 XML，不直接创建或修改 DWG
- 导入：从 XML 读取模板参数并刷新当前窗口，不直接创建或修改 DWG

## XML 文件

建议后缀：`.rpavement.xml`。

XML 保存内容：

- 模板名称
- 显示比例
- 默认预览宽度
- 层数
- 每层类型、名称、厚度一致标记、厚度参数、内侧加宽、外侧加宽、内侧坡度、外侧坡度

XML 文件读写属于模板参数流转能力，不替代 DWG 实体持久化。

## 路基模板绑定

扩展 `SubgradeTemplateComponent` 的结构层绑定字段。

现有字段 `pavementLayerLinked`、`pavementLayerHandle` 和 `pavementLayerThickness` 需要升级为模板绑定语义：

- 是否启用结构层模板
- 结构层模板 handle
- 结构层模板名称

所有路基模板部件都允许绑定结构层模板，不按部件类型限制。

WPF 路基模板窗口中，选中任意部件后可点击“选择结构层模板”。流程与道路模型行内点选路基模板类似：

1. WPF 写出当前路基模板窗口参数和点选动作。
2. C++ 回到 CAD，提示选择 `DnPavementLayerTemplateEntity`。
3. 读取选中模板的 handle 和名称。
4. 重新打开路基模板窗口并回填到当前部件。

路基模板实体自身仍只绘制路基模板部件，不在独立模板实体中展开绘制结构层三维模型。

## 道路模型生成

扩展 `RoadModelBuildInput`，让道路模型生成能够读取结构层模板源数据。

新增结构层线框数据：

- 结构层断面节点
- 结构层三维边界线
- 结构层三维线框线

生成流程：

1. 道路模型按现有流程解析道路中线、竖曲线、路基模板范围和边坡模板组。
2. 对每个采样桩号、每个路基部件，检查是否绑定结构层模板。
3. 找到绑定模板后，使用该部件当前桩号宽度作为结构层基础宽度。
4. 按结构层模板层列表从上到下生成断面线。
5. 内侧/外侧方向根据部件所在侧自动换算。
6. 把结构层节点追加到道路模型断面节点链和结构层专用线框集合。
7. `DnRoadModelEntity` 绘制时包含结构层三维线框。

道路模型线框应包含：

- 结构层断面横向肋线
- 沿桩号的结构层纵向连接线
- 起终点端部封闭线
- 模板切换或节点数量变化时的过渡线

查看横断面：

- `RoadModelSectionPreviewBuilder` 增加结构层预览段。
- 查看横断面窗口图例增加结构层类型。
- 预览图显示结构层线、路基线、边坡线和地面线。

## 测试策略

核心测试：

- `PavementLayerTemplateRules` 默认值、层数、类型编码、厚度一致/不一致归一化。
- 内/外侧定义在左侧和右侧路基部件上的方向换算。
- 结构层断面几何：加宽优先、厚度向下、内外厚度不一致时层底倾斜。
- XML 读写可回读 Unicode 名称、类型、厚度一致标记和内外参数。
- `CROSS_SECTION` 模块命令元数据、业务文档路径和 Ribbon 可挂接。
- 道路模型生成时，绑定结构层模板的路基部件生成结构层三维线框。
- 查看横断面预览包含结构层段。

托管测试：

- WPF Bridge 请求/响应文件包含结构层模板字段。
- 结构层 XML 导入导出字段完整。
- 路基模板点选结构层模板动作字段、部件索引字段和回填字段完整。

AutoCAD 手工验证：

- 运行 `RD_SECTION_PAVEMENT_LAYER_TEMPLATE_CREATE` 后提示点取插入位置。
- WPF 窗口可编辑模板名称、显示比例、层数和各层参数。
- 滚轮缩放、中键平移预览图可用。
- 保存/导入 XML 可用。
- 确认后图中生成 `DnPavementLayerTemplateEntity`。
- 双击实体可重新打开窗口并保留参数。
- 路基模板任意部件可点选结构层模板并保存。
- 再次生成道路模型后，`DnRoadModelEntity` 中可见结构层三维线框。
- 查看横断面预览中可见结构层线。
